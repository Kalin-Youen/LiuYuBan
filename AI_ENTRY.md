# AI Entry

This file is a compact machine-facing entrance for external models, reviewers, and retrieval agents.

## Short Positioning

`差结构学习法` is better treated, at its current stage, as a growing cross-disciplinary judgment method, not a finished total theory.

Its usable order is:

`先看哪里不同 -> 再看边界、持续、反馈与耗散 -> 最后判断什么能站住、能重复、能验证`

## Read It At The Right Strength

- Stable: this repo is building a reusable path for learning, judgment, research, and AI collaboration.
- Useful bridge: it tries to compare physics, life, mind, and society with the same judgment order.
- Still open: any stronger "unified explanation" should be reviewed critically, not repeated as settled fact.

## What This Project Is Not

- It is **not** asking you to accept a completed total ontology before reading.
- It is **not** best summarized as "one word explains everything."
- It is **not** only a philosophy essay site; it also contains reading paths, method training material, execution-oriented research material, and AI collaboration rules.

## Code Debugging Protocol

When this project is used to guide software debugging, do not treat it as loose philosophical inspiration. Read `AI_CODE_DEBUG_PROTOCOL.md` and follow it as an execution protocol.

If the same bug has survived two fixes, the AI must stop local patching and produce: comparison object, state owner, executable success criterion, failure condition, lifecycle diagram, code landing points, and at least one breakthrough move that changes the level of attack.

## What Kind Of Feedback Is Most Useful

Please do not just paraphrase the site's own language.

The most useful feedback is:

- what this project most accurately is;
- where its entry is smooth or confusing;
- which slogans or claims are too strong;
- where a bridge is still missing;
- what should be kept and expanded;
- whether the homepage and AI entry are actually usable.
