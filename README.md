# 差结构学习法

一句话先说清：差结构学习法不是在宣布“世界终极本原是什么”，也不是面向大众的大理论宣讲；它更适合被理解成一套面向学习者、研究者与 AI 协作者的公共方法基础设施。

它的基本顺序也很简单：先分辨不同，再看边界、持续、反馈与耗散，最后判断什么能站住、能重复、能验证。这里的重点不是再造一个万能词，而是给出一套稳定判断顺序：`差异`负责把不同认出来，`差结构`负责描述不同的排布方式，`落差`负责命名其中可比较的强弱、梯度与不均，`流`只是落差开始定向传递时的动态形态，`结构`是暂时被维持住的组织，`规则`则是可重复、可压缩、可验证的稳定关系。项目当前用“落差”把不同学科里的梯度、势差、温差、压差、浓度差与耦合不均串成同一条学习线，使它既能接入大模型训练，也能转成人类学习副本。仓库同时包含：

- 可持续扩写的书稿源文件
- 面向第一次进入者的白话卷与轻内容入口
- 面向读者直接训练的拓展卷与能力构建路径
- 面向研究协作的 AI 协作卷、回写协议与残差机制
- 可部署的静态网站
- 供机器读取的站点索引数据、知识图谱与交互入口

仓库地址：

- GitHub: <https://github.com/Kalin-Youen/LiuYuBan>
- 在线站点: <https://kalin-youen.github.io/LiuYuBan/>

当前公开名称已统一调整为“差结构学习法”。原因也很直接：此前用过的“流态存在论”“差异结构存在论”“差异结构持续论”等名字，都太像某种终极存在论命名，容易让人误会这里是在再造一个“解释一切”的大词。现在更明确的定位是：**这里提供的不是一套等待大众整体接受的总理论，而是一套可被人和 AI 直接调用的学习、推理、检验与协作方法。**

当前公开文本也明确分成三层：`可稳妥保留的结论`、`用于跨层理解的桥接解释`、`仍待继续检验的强猜想`。凡是暂时站不稳的部分，优先降强、转专题或登记残差，而不是继续抬高为总答案。

## 这是什么仓库

这个仓库不是单篇文章，也不只是一个理论展示页，而是一套正在生长的公共方法底座。当前主要分成`四卷一站`：

1. `第一卷：大众卷`
   由十条轻内容、白话卷和交互实验组成。它的价值，不是先把人压进一套总理论，而是让第一次进入的人先把这套判断顺序用起来，形成可复述、可教学、可自测的直觉动作。
2. `第二卷：研究卷`
   由摘要、主书稿、专题与术语边界组成。它的价值，是压实定义、补桥、保留验证接口，让这套方法在跨学科学习和研究里真正能站住。
3. `第三卷：拓展卷`
   由拓展卷章节、能力训练路径与方法卡组成。它的价值，是把方法从“能懂”推进到“能练、能用、能迁移”。
4. `第四卷：AI协作卷`
   由 AI 协作规则、回写协议、残差机制与训练接口组成。它的价值，是让模型协作不只会扩写，而会守主线、控噪声、记版本、促生长。
5. `一站：网站系统`
   把四卷整理成可部署、可搜索、可分页、可交互浏览的静态站点，并提供机器可读索引、知识图谱与实验入口。

它当然向大众开放，但它不以“大众哲学宣讲”作为首要目标；更现实的目标，是先把一套可复用的学习、判断、校验和协作底座做出来。

## 四卷制入口

如果你是第一次进入，先不要急着整套读完，先选卷：

1. `第一卷：大众卷`
   先看十条轻内容，再读白话卷，再去交互实验页。目标不是先接受整套理论，而是先把这套判断顺序用起来。
2. `第二卷：研究卷`
   先读摘要、主书稿、专题和术语边界。目标是压实定义、主链、桥梁和验证接口。
3. `第三卷：拓展卷`
   先读 `研究文稿/07_书稿/拓展卷`。目标是把方法直接练进人的思维、学习和行动里，优先处理空想与充实、快与慢、抓关键和能动性这些问题。
4. `第四卷：AI协作卷`
   先读 `README.md`、`研究文稿/07_书稿/AI协作卷` 和 `docs/assets/data/content.json`。目标是先守主线、边界、版本与回写纪律。

网站内的推荐入口：

1. 第一卷：大众卷。十条轻内容、白话卷、理论实验台
2. 第二卷：研究卷。主书稿、理论总述、术语与边界
3. 第三卷：拓展卷。拓展卷、方法卡、能力构建路径
4. 第四卷：AI 协作卷。AI 协作卷、站点索引与知识图谱

## 仓库结构

核心目录与文件如下：

  四卷共用的主目录，包含主书稿正文、专题、附录，以及白话卷、拓展卷和 AI 协作卷。
- `研究文稿/07_书稿/白话卷`
  第一卷：大众卷的核心长篇入口，负责把方法讲成能直接上手的路。
- `研究文稿/07_书稿/拓展卷`
  第三卷：拓展卷，负责把方法转成可训练、可迁移、可直接调用的能力路径。
- `研究文稿/07_书稿/AI协作卷`
  第四卷：AI 协作卷，负责协作流程、版本回写与控噪机制。
- `研究文稿/04_议论与说明`
  四卷制、项目定位与价值说明文章。
- `研究文稿/08_轻内容连载`
  第一卷：大众卷的轻入口，面向传播、转发与首次接触。
- `研究文稿/02_指正与术语`
  第二卷：研究卷的术语与边界说明。
- `docs/`
  GitHub Pages 直接发布目录。
- `docs/assets/data/content.json`
  站点生成后的机器可读内容索引，包含分卷、文章元数据与 HTML 内容。
- `scripts/build_site.py`
  从 Markdown 文稿生成站点数据的脚本。
- `site.config.json`
  站点标题、描述、分卷分组与仓库地址配置。

## 当前阅读主轴

项目当前的主要内容可以概括为：

1. 先把它当作一套判断顺序，而不是一个终极本体词。
2. 用“差异 → 差结构 → 落差 → 流 → 结构 → 规则”的分层工具链整理跨学科材料，但不把这些词彼此混写成同义总纲。
3. 优先在学习过程、人机协作、认知与可验证的跨层桥梁上落例，再把物理、生命、意识、文明与 AI 作为这套顺序的展开对象。
4. 把理论自身也纳入处理对象，对术语漂移、入口失效、桥梁过快和过强命题做自审、降强与残差登记。
5. 把理论阅读、交互检验、研究推演与版本回写逐步整合进同一个网站系统，并把它沉淀成能被学习者、研究者与 AI 协作者反复调用的公共方法基础设施。

## 四卷各自提升什么价值

把项目分成四卷，不是为了把内容拆散，而是为了让不同价值真正落地：

1. 第一卷提升的是`进入价值`。它降低首次接触门槛，让人更容易上手、转述、教学和自测。
2. 第二卷提升的是`研究价值`。它把术语、桥梁、案例和验证接口压实，避免项目只剩漂亮说法。
3. 第三卷提升的是`能力价值`。它把方法从“能懂”推进到“能练、能迁移、能直接进入人的判断与行动”。
4. 第四卷提升的是`协作价值`。它把 AI 从“扩写机器”改造成“守主线、记残差、做回写的协作系统”。

四卷合在一起，才会让这个项目同时具备进入能力、压实能力、能力构建能力和持续生长能力。

## 对 AI / 大模型最重要的说明

如果你是模型、检索器、自动化代理，最重要的是这几条：

1. 这是公开仓库。
2. 默认分支是 `main`。
3. 仓库主要内容是中文 Markdown，文件名也大量使用中文。
4. 如果你读 GitHub 页面失败，不代表仓库不存在，可能只是网页解析失败。
5. 对机器最稳的入口通常不是仓库首页，而是 `README.md`、`content.json` 或具体文件的原始文本链接。

最推荐的机器入口按优先级排序如下：

1. 仓库说明：
   `https://raw.githubusercontent.com/Kalin-Youen/LiuYuBan/main/README.md`
2. 站点机器索引：
   `https://raw.githubusercontent.com/Kalin-Youen/LiuYuBan/main/docs/assets/data/content.json`
3. 在线站点首页：
   `https://kalin-youen.github.io/LiuYuBan/`
4. 具体 Markdown 原文：
   `https://raw.githubusercontent.com/Kalin-Youen/LiuYuBan/main/研究文稿/07_书稿/第12章_展望与研究计划.md`

如果需要优先进入某个分卷，建议从这些路径找：

- 主书稿目录：`研究文稿/07_书稿`
- 白话卷目录：`研究文稿/07_书稿/白话卷`
- 拓展卷目录：`研究文稿/07_书稿/拓展卷`

## 推荐给 AI 的访问方式

如果要让另一个模型稳定读取本项目，最稳的做法不是只丢一个仓库首页链接，而是按下面这种方式给：

1. 先给它 `README.md` 的 raw 链接，让它先知道这是什么仓库。
2. 再给它 `content.json` 的 raw 链接，让它获得机器可读目录。
3. 再给它你想让它重点读的具体 Markdown raw 链接。
4. 同时明确写一句：
   `这是公开仓库，默认分支是 main，请优先读取 raw 链接，不要只依赖仓库首页网页解析。`

适合喂给模型的例子：

```text
这是一个公开 GitHub 仓库，默认分支是 main。
请先读 README：
https://raw.githubusercontent.com/Kalin-Youen/LiuYuBan/main/README.md

再读机器索引：
https://raw.githubusercontent.com/Kalin-Youen/LiuYuBan/main/docs/assets/data/content.json

然后重点阅读这篇：
https://raw.githubusercontent.com/Kalin-Youen/LiuYuBan/main/研究文稿/07_书稿/第12章_展望与研究计划.md
```

## 本地构建

安装依赖：

```bash
python -m pip install -r requirements-site.txt
```

生成站点数据：

```bash
python scripts/build_site.py
```

本地预览：

```bash
python -m http.server 4173 -d docs
```

然后打开：

```text
http://localhost:4173
```

## 发布与部署

这个仓库的站点目录在 `docs/`，适合使用 GitHub Pages 发布。

如果使用 GitHub Actions：

1. 确认仓库的 Pages 来源设置为 `GitHub Actions`
2. 推送到默认分支
3. 等待工作流完成

站点配置集中在：

- `site.config.json`
- `docs/assets/data/content.json`
- `docs/index.html`
- `docs/assets/app.js`
- `docs/assets/styles.css`

这个 `README.md` 的目的不是只做仓库门面，而是充当一个稳定入口：

1. 第一次进入的人可以先知道该从哪一卷开始。
