# 第4章 高中学习诊断系统：把稳态窗口、失稳判别与复习调度做成网站

上一章把“稳态窗口—失稳判别—改组转层”直接压到了高中生自己的学习动作里。  
这一章再往前走一步，不再只问“学生自己怎么用”，而是问：

> **能不能把这套判断顺序做成一个真正可操作的网站系统，让拍照、错题、知识点、复习计划和状态诊断接成一个闭环？**

这里先把边界说清：  
这一章不是在宣称产品已经成立，更不是在宣称理论已经被教育场景证明；它只是把当前最值得优先推进的一条应用线，压成一份更像产品方案和验证路径的文稿。

---

# 1. 为什么优先选高中学习

如果这套方法后续真要接受检验，高中学习是当前很适合先落的一条线。原因不是“教育天然高尚”，而是这个场景同时具备了几个很难得的条件：

1. 有明确目标：分数、正确率、知识点掌握度都能落成指标。
2. 有稳定周期：日练、周测、月考、期中、期末天然形成多层时间窗口。
3. 有清晰阈值：及格线、目标分、题型正确率、限时完成度都能作为门槛。
4. 有可见耗散：遗忘、手感下降、知识点混淆、疲劳都能被直接观察。
5. 有明确补给：复习、讲解、重做、专题修复、睡眠恢复都能进入调度。
6. 有失稳现象：连续错同类题、连续掉分、恢复缓慢都很容易被记录。
7. 有改组需求：很多“大问题”最后都需要被重切成更小的错因和题型。
8. 有转层问题：作息、压力、计划、科目分配会直接改写底层表现。

也正因为这些条件同时存在，高中学习不是一个随便找来的例子，而更像是：

> **一条把“稳态窗口/失稳诊断”从解释语言推到可记录、可调度、可比较对象的实战线。**

---

# 2. 产品定位不能只是“AI 错题本”

如果只按普通学习工具做，这个系统的流程通常会写成：

```text
拍照 -> 识别题目 -> 收集错题 -> 给解析
```

这条线当然也有价值，但它还不够体现本项目的独特判断力。  
更贴近这一套框架的写法应当是：

```text
拍照 -> 识别题目与作答 -> 抽取知识点与题型 -> 判断错因
-> 计算状态 -> 生成复习窗口 -> 安排复测 -> 追踪恢复
```

因此它更合适的定位，不是“拍照搜题网站”，而是：

> **一个基于失稳诊断的学习诊断与复习调度系统。**

它真正想解决的，不是“再多收集一点错题”，而是下面这些更硬的问题：

```text
哪里只是短期波动？
哪里已经连续失稳？
哪里是分类方式太粗，需要改组？
哪里不是知识点问题，而是更高层的学习节奏问题？
下一轮该复习什么？
多久后必须复测？
```

---

# 3. 主闭环：从拍照到复测

当前最值得先压实的主闭环，可以直接写成：

```text
上传试卷 / 作业照片
-> OCR 与题目切分
-> 识别标准答案与学生作答
-> 判断对错
-> 抽取知识点、题型与难度
-> 生成错题卡
-> 判断状态
-> 安排复习与复测
-> 记录恢复结果
-> 更新知识点状态图
```

这一闭环里，真正关键的不是“OCR 有多聪明”，而是中间三步能不能站住：

1. **错因是否被压到了最小失稳点。**
2. **状态是否被判成波动、失稳还是改组/转层问题。**
3. **复习计划是否真的按窗口而不是按直觉发任务。**

如果这三步做不稳，就算拍照识别再顺，最后也只是把旧式错题本自动化了一层。

---

# 4. 错题卡必须从“解析卡”升级成“诊断卡”

普通错题本最常见的结构是：

```text
题目
答案
解析
```

这对查漏有帮助，但不足以支持状态判断。  
更稳的写法是把每道错题压成一张结构化诊断卡，例如：

| 字段 | 示例 |
|---|---|
| 题目来源 | 2026 高二数学周测 |
| 科目 | 数学 |
| 章节 | 导数 |
| 知识点 | 恒成立 / 分类讨论 |
| 题型 | 参数范围 |
| 错误步骤 | 不会分离参数 |
| 错因 | 方法选择错误 |
| 连续出现次数 | 第 3 次 |
| 当前状态 | 正式失稳 |
| 建议动作 | 暂停混刷，转入专题修复 |
| 下次复测 | 2 天后 |

这里最关键的一步，是把“错题”翻译成“状态样本”。  
一旦这样写，错题卡就不再只是存档，而开始承担两个作用：

1. 给学生看清自己错在什么最小位置。
2. 给系统累计判断某个知识点是否正在穿阈。

---

# 5. 状态机要先写清，再谈智能化

第一版系统不应一上来就追求复杂模型，而应先把状态机写清楚。  
当前最稳的首轮状态，建议至少分成下面六种：

| 状态 | 含义 | 默认动作 |
|---|---|---|
| 正常 | 仍在窗口内 | 维持当前节奏 |
| 波动 | 单次或短期穿线 | 记录并观察下一轮 |
| 失稳候选 | 连续两次低于最低线，或恢复变慢 | 缩短复测间隔 |
| 正式失稳 | 连续三次穿线，或多轮恢复不上来 | 停止盲刷，进入专题修复 |
| 已改组 | 原分类过粗，已拆成更细题型/错因 | 用新分类继续跟踪 |
| 恢复中 | 已脱离最深谷底，但尚未稳定 | 保持短周期复测 |

如果把它再压成最简判据，可以先用：

```text
低于最低线 1 次 = 波动
连续 2 次 = 失稳候选
连续 3 次 = 正式失稳
```

这组阈值不是永恒常数，只是第一轮执行规则。  
它的意义不在于“数学上最优”，而在于先把状态判定从情绪语言里抽出来。

---

# 6. 最小失稳点比“大科目焦虑”更重要

很多学生会说：

```text
我数学差。
我英语不行。
我物理崩了。
```

但这类说法太大，不能直接进入修复。  
系统真正有价值的地方，是把大焦虑压回最小失稳点，例如：

```text
不是“数学差”，而是“导数恒成立中的参数分离失稳”。
不是“英语差”，而是“阅读推断题中过度推断失稳”。
不是“物理差”，而是“电磁感应中的过程分段失稳”。
```

因此错因分类不能只停在“会/不会”，而应按学科至少拆到下面这种层级：

## 数学常见错因

- 概念不清
- 公式不熟
- 题型识别失败
- 方法选择错误
- 计算错误
- 分类讨论遗漏
- 步骤表达不完整
- 时间不足
- 审题错误

## 英语常见错因

- 单词不认识
- 熟词生义
- 长难句断句失败
- 主旨误判
- 细节定位错误
- 过度推断
- 选项陷阱
- 固定搭配不熟

## 物理常见错因

- 对象选择错误
- 受力分析错误
- 过程分段错误
- 公式适用条件错误
- 方向判断错误
- 图像理解错误
- 单位错误
- 守恒条件误用

第一版系统没必要把分类做到极细，但必须守住一个原则：

> **一旦用户说出“我整科不行”，系统要能继续往下压，而不是接受这句大话作为最终诊断。**

---

# 7. 复习计划的核心不是“多做”，而是“按窗口调度”

普通计划常写成：

```text
明天复习错题。
本周刷两套卷。
每天背 30 个单词。
```

这不是不能用，而是太粗。  
更贴近当前框架的写法，应当由状态来决定动作：

## 波动

```text
先加入观察列表
不大幅增加任务
下一轮同类题出现时再判断
```

## 失稳候选

```text
2 天内安排 3 道同类题复测
要求记录错因
必要时补一轮基础题
```

## 正式失稳

```text
停止混刷
进入专题修复
第 1 天：看讲解
第 2 天：基础题
第 3 天：中档题
第 5 天：复测
第 7 天：混合检测
```

## 改组

```text
重切知识点和题型
拆出新的子类错题卡
按新分类重算状态
```

这一层如果要进一步落成计算字段，首轮已经够用的量通常有：

```text
p0：刚复习后的掌握度
lambda：遗忘/退化速度
pcrit：最低保持线
Pi_max：最大复习间隔
status：当前状态
```

第一版不必急着拟合复杂公式。  
只要先守住下面这条规则，系统就已经比“凭感觉复习”更硬：

```text
连续答对 -> 延长间隔
答错一次 -> 观察
连续答错 -> 缩短间隔
正式失稳 -> 进入专题修复
恢复后 -> 重新估计间隔
```

# 8. 知识图谱要先做成“个人学习健康地图”

这一块很容易一上来做大，但第一版不必追求完整教材知识图谱。  
更稳的起点是三层叠加：

1. 教材章节树
2. 考点/题型树
3. 学生个人状态标记

例如数学“导数”节点下，可以显示：

```text
导数
├── 求导运算：绿色
├── 单调性：绿色
├── 极值最值：黄色
├── 恒成立：红色
├── 分类讨论：红色
└── 切线问题：橙色
```

颜色建议也尽量服务于状态，而不是装饰：

- 绿色：稳定
- 黄色：波动
- 橙色：失稳候选
- 红色：正式失稳
- 蓝色：恢复中
- 灰色：未检测

这样学生看到的不是一张大而空的脑图，而是：

> **一张自己的学习健康地图。**

# 9. 仪表盘、周复盘和老师/家长视图

如果系统只停在“收错题”，用户很难获得连续反馈。  
因此至少还应补三个输出层。

## 9.1 学生仪表盘

建议优先展示：

```text
本周波动点
本周失稳点
正在恢复的知识点
超过复习窗口的知识点
即将掉出窗口的知识点
高层节奏风险
```

## 9.2 周复盘报告

重点不该是“你本周做了多少题”，而应是：

```text
新增波动点多少
正式失稳点多少
已恢复知识点多少
超过复习窗口的点多少
下周最优先修哪 1-3 个点
```

更重要的是结论句要避免大而空，而要像这样：

> 本周主要问题不是数学整体下降，而是导数恒成立和解析几何计算连续穿线。  
> 导数恒成立已进入正式失稳，建议下周暂停混刷，转入 3 天专题修复。

## 9.3 老师/家长视图

这两类视图应当服务于定位，而不是监控。

老师更需要看：

```text
全班哪个知识点普遍失稳
哪些学生只是波动，哪些已经正式失稳
哪个题型值得重新讲
```

家长更需要看：

```text
孩子当前最需要处理的 1-3 个问题
是否是作息/计划层问题
下周建议怎样配合
```

如果这层做得对，系统就不只是给学生用，也会减少老师和家长对“整体退步”的误判。

# 10. 第一版一定要允许人工确认

拍照识别试卷看起来最像 AI 能大显身手的地方，但第一版越要谨慎。  
原因很简单：这里最容易连续出错的是：

1. OCR 识别错题干
2. 标准答案抓错
3. 学生作答切分错
4. 知识点标错
5. 错因判断错

因此更稳的第一版不应写成“全自动诊断”，而应写成：

```text
AI 先生成
用户再确认或修改
系统记录确认后的干净数据
```

尤其是错因这一步，第一版甚至更适合让学生从一组选项里点选，例如：

```text
我不会
算错
审题错
公式错
时间不够
没想到方法
```

这不是降低智能，而是在为后续模型积累可用标签。

# 11. MVP 不该做太大

当前最值得优先落的 MVP，可以只做四件事：

## MVP 1：拍照上传与题目识别

支持：

```text
拍照
OCR
AI 识别题目与作答
用户确认科目/答案/错因
```

## MVP 2：错题结构化

生成：

```text
知识点
题型
错因
难度
状态
下次复测时间
```

## MVP 3：失稳判定

按连续错误和恢复情况，至少区分：

```text
波动
失稳候选
正式失稳
恢复中
```

## MVP 4：复习计划

每天输出：

```text
今天必须复习的错题
今天需要复测的知识点
今天不建议继续盲刷的失稳点
```

只做这四个模块，已经足以开始验证：

> **系统是不是比普通错题本更早发现问题、并更快推动恢复。**

# 12. 第一阶段主攻科目：先做高中数学

如果一上来做全科，很容易什么都有一点，但什么都不够硬。  
第一阶段更稳的主攻，仍然是：

> **高中数学错题诊断 + 复习计划。**

原因有七条：

1. 题型结构清晰。
2. 对错边界明确。
3. 错因较容易分类。
4. 连续失稳容易观察。
5. 家长和学生痛点足够强。
6. 付费意愿通常高于很多泛学习功能。
7. 最适合验证“连续穿阈”和“专题修复”的效果。

首批甚至可以再收窄到：

```text
函数
导数
解析几何
数列
```

先把一个科目做穿，比全学科浅铺更符合当前阶段的主攻方向。

## 前端

- React / Next.js

## 后端

# 14. 真正要验证的不是“酷不酷”，而是四组指标

这条产品线最重要的地方，是它可以把理论推进到可检验状态。  
因此系统上线后，不该只看 DAU、上传量，而应优先看四组指标：

## 14.1 正式失稳点是否减少

例如：

```text
连续 3 次错同类题的知识点数量是否下降？
```

## 14.2 复习窗口是否能预测遗忘

例如：

```text
超过系统建议复习间隔的知识点，错误率是否明显上升？
```

## 14.3 改组是否提高修复效率

例如：

```text
把“大类错题”拆成最小失稳点后，恢复速度是否加快？
```

## 14.4 失稳预警是否早于考试掉分

例如：

```text
系统提前发现导数恒成立失稳后，是否能预测下一轮相关掉分？
```

这四组指标如果站住，这个系统就不仅是学习工具，也会反过来成为理论验证接口。

# 15. 边界与注意事项

这一线虽然值得推进，但也有几个边界必须先守住。

## 15.1 不要把教材原文直接当数据仓

更稳的方式是：

1. 优先存知识点结构，不直接存整本教材正文。
2. 引用章节名、页码和题型，不大段复制原文。
3. 允许用户上传自己的试卷、笔记和教材页做个人解析。
4. 优先接公开课标、考试大纲、开源题库。

## 15.2 不要把 AI 判断写成最终结论

AI 在第一版更适合做候选生成和结构整理，而不是最后裁决。  
凡是题目识别、知识点归因、错因判断，都应允许人工修改。

## 15.3 不要一上来就做“全能学习平台”

当前这条线最怕的，不是功能不够多，而是主线被泛化。  
更稳的顺序仍然是：

```text
先做一个科目
先做一个闭环
先证明失稳诊断比普通错题本更有用
再往外扩
```

# 16. 这条产品线的真正差异是什么

市面上已经有很多产品在做：

```text
拍照搜题
AI 解题
错题本
知识点推荐
学习计划
```

这条线真正不同的地方，不在于也有这些功能，而在于它看问题的顺序不同：

> 别人记录错题，这里判断失稳。  
> 别人推荐题目，这里安排恢复窗口。  
> 别人盯着总分，这里盯连续穿阈。  
> 别人默认“再多刷一点”，这里会告诉学生什么时候该停下来重切。

如果这一点做不出来，它就只是另一种 AI 学习工具。  
如果这一点做出来，它才配得上“稳态窗口/失稳诊断系统”这个名字。

---

# 17. 本章最后压成一句

高中生真正缺的，往往不是努力本身，而是：

> **不知道自己的学习系统到底哪里只是在波动，哪里已经失稳，哪里需要改组，哪里已经不是低层题目问题，而是更高层节奏需要重排。**

所以，这条网站方案最值得先做的，不是把功能堆满，而是先把下面这条闭环做实：

```text
拍照识别错题
-> 结构化知识点与错因
-> 连续穿阈判断
-> 生成复习窗口
-> 安排复测
-> 追踪恢复
-> 更新学习状态图
```

只要这条线先站住，后面的知识图谱、老师视图、周复盘、AI 讲解和理论验证后台，都会有地方接上。  
反过来，如果这条主闭环站不住，其余功能只会把系统做得更热闹，而不会更有判断力。
