# 第2章 首轮样本验证：从公式卡到物理、学习与 AI 协作样本

第 1 章到 1.9 为止，职责已经很清楚了：公式先冻结，不再继续长枝。接下来真正该推进的，不是再推一个更一般的表达式，而是让已经冻结下来的窗口核开始接受样本、对照和失败记录的检验。

因此，本章不再承担“继续推导”的任务，而承担三件更硬的事：

1. 把第 1 章冻结下来的公式卡压成可执行验证对象；
2. 让物理、学习与 AI 协作三类样本进入同一套验证程序；
3. 把支持、失败、残差和升级条件分开登记，而不是混成一句“整体看起来还行”。

若只想看“差异思维与现有学习策略”的并排比较，那条线已经从本章主位置移出，单独保留为专题稿：`专题_差异思维与现有学习策略的比较_当前最硬结论与下一步验证.md`。这里的主线更窄，也更硬：只问第 1 章冻结下来的窗口模型，到底能不能在样本里站住。

## 2.1 本章要验证什么

当前冻结下来的可用对象，不是所有占位式、母式和广义候选式，而是三层最小结构。

### 2.1.1 最小骨架

$$
\Delta_{\mathrm{lim}}^{\mathrm{exact}}(\Pi,\lambda)
=
\frac{K\Pi}{e^{\lambda\Pi}-1}.
$$

它只负责说明一件事：在周期补给与指数耗散共同作用下，连续幅值系统会出现一个随周期拉长而降低的局部承受上界。

### 2.1.2 两张冻结公式卡

物理 / 连续幅值支路：

$$
\Delta_{\mathrm{win}}(\Pi\mid z)
=
\frac{K_1(z)e^{-\tau_f(z)/\Pi}}
{e^{\lambda(z)\Pi}-1},
\qquad
\Pi\ge \tau_h(z).
$$

学习 / 概率有界支路：

$$
p_{\mathrm{win}}(\Pi\mid z)
=
p_0(z)\exp\!\left[
-\lambda(z)\Pi
-\frac{\tau_f(z)}{\Pi}
\right],
\qquad
\Pi\ge \tau_h(z).
$$

### 2.1.3 执行判据

$$
G(\Pi\mid z)\ge G_{\mathrm{crit}}
\quad\Longleftrightarrow\quad
\Pi\in \mathcal{I}(z)\cap[\tau_h(z),+\infty).
$$

因此，本章不是再问“公式还能不能更强”，而是先问：

1. 哪些参数能直接观测；
2. 哪些参数只能反算；
3. 哪些量目前只能做代理；
4. 哪些支路在什么样本上开始失效；
5. 这些窗口核是否优于更简单的替代模型。

## 2.2 本章的验证纪律

若没有统一纪律，三类样本最后只会变成三套互不相认的故事。当前更稳的做法，是先把验证程序写死。

### 2.2.1 先切局部状态，再谈公式

所有验证都必须先固定局部切片 `z`。这里的 `z` 不只是一个形式变量，而是把协议、历史状态、边界条件、资源状态、对象层级和可用反馈都收进去的局部状态。

这意味着：

1. 不把不同协议的物理数据直接混拟合；
2. 不把不同 `history / language pair / lexeme` 的学习数据直接混拟合；
3. 不把不同审稿负担、任务复杂度和回写密度的 AI 协作日志直接混拟合。

若切片没有先做，公式看上去再整齐，也更像被平均抹平后的幻觉。

### 2.2.2 先分清观测量、反算量与代理量

每个样本进入验证前，至少要把变量分成三类：

1. **可观测量**
   可以直接从记录、协议或原始数据中读出；
2. **反算量**
   需要由拟合或边界反解得到；
3. **代理量**
   目前不能直接解释为本体变量，只能在给定协议和局部切片下充当有效近似。

只有把这三类分开，后面的比较、残差和失败记录才有意义。

### 2.2.3 不只看拟合，还要看边界预测

本章不接受“能拟合就算站住”这种写法。真正要看的，至少有四类输出：

1. 参数是否稳定；
2. 左右边界是否可重复预测；
3. 失稳前兆能否提前出现；
4. 模型外推是否比简单经验式更稳。

所以，本章的第一原则不是追求最小残差，而是追求**边界判断是否可靠**。

### 2.2.4 失败样本必须与成功样本并列登记

若只收成功样本，这套模型就会越来越像什么都能解释。为避免这一点，本章要求：

1. 每类样本至少登记一组支持样本；
2. 每类样本至少登记一组失败样本或高残差样本；
3. 失败不被当成“暂时数据不好”，而先被当成边界提示。

### 2.2.5 先把四张治理表写死

到这里如果还只写原则，本章仍然容易在后面的样本部分重新滑回“边做边改口”的旧路。  
因此，在正式进入物理、学习与 AI 协作三条样本线之前，先把当前版本最需要的四张治理表写死。它们不是终局定论，而是本章的**工作版纪律板**：后面所有支持卡、失败卡、对照卡，都默认在这四张表的约束下登记。

## 2.3 物理样本：验证连续幅值窗口核

物理线当前要验证的，不是“世界是否最终服从这条式子”，而是更窄的命题：

> 在局部协议与局部状态切片内，连续幅值窗口核能否比更简单写法更稳地给出可行区间、边界收缩和失稳前兆。

### 2.3.1 当前要收的量

物理样本至少要收六类量：

1. 周期量 `\Pi`；
2. 连续幅值代理 `\Delta_{\mathrm{proxy}}`；
3. 局部有效耗散率 `\lambda(z)`；
4. 快周期软尺度 `\tau_f(z)`；
5. 硬下界 `\tau_h(z)`；
6. 支撑代理量 `K_1(z)` 或 `K_{\mathrm{proxy}}(\Pi,z)`。

其中：

1. `\lambda(z)` 应尽量从无补给衰减段或近似无补给区间中识别；
2. `\tau_f(z)` 优先从快周期端掉效开始出现的位置估计；
3. `\tau_h(z)` 优先从协议约束、最小恢复时间或最小结算时间读取；
4. `K_1(z)` 在当前阶段仍主要被视为有效支撑代理量，而不是已完全拆开的纯注入常数。

### 2.3.2 当前要验的四件事

1. **窗口是否单峰**
   若快周期惩罚接入后，窗口仍不呈现单峰或局部双边界结构，就说明当前核对该切片未必合适。
2. **边界是否可预测**
   用局部参数估计出的 `\Pi_{\min}, \Pi_{\max}` 是否能与样本中的失稳区、可行区大致对上。
3. **近峰坍缩是否出现**
   当阈值逼近峰值时，`W^2/(1-\rho)` 是否开始趋于稳定。
4. **参数是否稳定**
   同一局部协议下，参数是否能在误差范围内保持同量级，而不是每次都漂到另一套解释。

### 2.3.3 物理线的失败标记

若出现下列情况，当前物理卡应先判为未站住，而不是继续往里补项：

1. `\lambda(z)` 无法从局部衰减段稳定识别；
2. 快周期端没有明显掉效，`\tau_f(z)` 失去进入必要性；
3. 窗口不呈现单峰结构；
4. `K_{\mathrm{proxy}}` 的变化主要由协议切换而非周期变化主导；
5. 近峰区 `W^2/(1-\rho)` 系统性漂移。

### 2.3.4 首张实填验证卡：Battery Pulse Diagnostics 的 `8s \rightarrow 50s`

到这里如果还只停在“该收什么量”，本章仍然偏空。当前最适合先落成一张实填卡的，不是自造 RC 样本，而是已经能被公开脚本直接复出的 `Battery Pulse Diagnostics`。原因很简单：它虽然还不够干净到承担“最终定律样本”，但已经足够承担“首轮公开验证卡”。

当前可直接挂接的复现编号有两条：

1. `PHY-BPD-2C-01`
   负责从公开特征表里压出 `8s` 与 `50s` 两个局部周期点上的 `\lambda_{\mathrm{eff}}` 与 `K_{\mathrm{proxy}}`；
2. `PHY-WIN-GRID-01`
   负责在 `\eta=1,\ q=1` 的首轮窗口核下，给出归一化阈值族对应的可执行区间。

若把它压成一张最小验证卡，当前更稳的写法可以直接记成：

```text
验证卡编号：PHY-BPD-2C-01
样本类型：公开电池脉冲诊断样本
局部切片：Battery Pulse Diagnostics / 2C / shared 75 cells
比较轴：Π = 8s（rapid 4s+4s） vs Π = 50s（HPPC 10s+40s）
观测量：δ0, δres, Trest
反算量：λ_eff, K_proxy
辅助输出：归一化窗口表、近峰宽度坍缩代理
当前用途：验证局部骨架、K 随 Π 塌缩、窗口核是否出现单峰边界
```

这张卡当前已经能填进下表：

| 周期 `\Pi` | 协议 | 共同 `cell` 数 | `\lambda_{\mathrm{eff}}` 均值 | `K_{\mathrm{per\,C}}` 均值 |
| --- | --- | ---: | ---: | ---: |
| `8s` | rapid `2C`, `4s+4s` | `75` | `0.532915 s^-1` | `0.014721` |
| `50s` | HPPC `2C`, `10s+40s` | `75` | `0.053926 s^-1` | `0.001664` |

只看这张卡，当前至少能压出三条首轮判断。

第一，**局部支撑代理 `K_{\mathrm{proxy}}` 不能再被写成与周期无关的全局常数。**  
在同一公开源、同一 `2C` 条件下，仅把局部周期从 `8s` 拉到 `50s`，`K_{\mathrm{hppc}}/K_{\mathrm{rapid}}` 的均值已经降到约 `0.1156`。若只用这两个点做首轮幂律压缩，得到的两点指数约为 `-1.19`。这还不够写成稳定定律，但已经足够把“`K` 可先当全局常数”降级掉。

第二，**`\lambda_{\mathrm{eff}}` 目前更像局部协议段上的有效参数，而不是跨协议不变的对象常数。**  
同一公开样本线里，`8s` 与 `50s` 两类协议给出的 `\lambda_{\mathrm{eff}}` 均值相差接近一个数量级。这意味着：即使对象层还是同一批电芯，协议切片不同，`z` 也已经被显著改写。

第三，**这张卡已经足以验证“物理线至少抓住了方向”，但还不足以单独完成整条窗口定律。**  
它已经支持“局部耗散可识别”“局部支撑会随 `\Pi` 塌缩”“窗口核值得继续保留”这三点；但它还不足以只靠自己就给出化学体系级通用指数、绝对阈值或普适 `\tau_h`。

同一批公开输出还给了一个很有用的辅助检查。当前 `public_physical_window_grid.csv` 用最快切片作参考，在 `BPD 2C proxy` 上给出了三组归一化阈值输出：

| 峰值分数 `\rho` | `\Pi_{\min}^{\mathrm{exec}}` | `\Pi_{\max}^{\mathrm{exec}}` | `W_{\mathrm{exec}}` | `W_{\mathrm{exec}}^2/(1-\rho)` |
| --- | ---: | ---: | ---: | ---: |
| `0.5` | `1.97s` | `6.53s` | `4.56s` | `41.57` |
| `0.8` | `2.54s` | `5.05s` | `2.51s` | `31.51` |
| `0.9` | `2.82s` | `4.53s` | `1.71s` | `29.29` |

这组数的价值，不是立刻宣称“真实物理阈值已经定了”，而是说明：**当阈值用峰值分数归一化后，窗口宽度确实在随 `\rho\uparrow 1` 收缩，且 `W^2/(1-\rho)` 已开始落到同量级。** 这为第 1 章的近峰坍缩式提供了首轮支持。

但这里也要把边界写死：这组窗口数值仍然是**归一化阈值族下的探索输出**，不是从独立物理失稳阈值反推出来的最终边界。换句话说，它目前更适合作为“窗口核形状检查”，还不适合作为“实验判死线”。

### 2.3.5 这张实填卡当前支持什么，不支持什么

若把上面的数收紧成可执行判断，那么这张卡当前支持的是：

1. 局部物理样本已经足以把 `\lambda_{\mathrm{eff}}` 与 `K_{\mathrm{proxy}}` 从空变量压成可复现输出；
2. `K` 随 `\Pi` 塌缩这一方向，在独立公开样本上已经再次出现；
3. 窗口核的近峰收缩，已经出现首轮数值支持；
4. 物理卡继续保留为第 1 章冻结公式的主支路，是有根据的。

这张卡当前还**不能**支持的是：

1. 只靠两点协议就写出稳定的 `K(\Pi)` 定律；
2. 把 `\lambda_{\mathrm{eff}}` 当作跨协议、跨状态不变的对象常数；
3. 把归一化峰值分数窗口直接当成真实物理可行区间；
4. 把电池脉冲这条线直接抬高成“连续幅值系统普遍已经证成”。

因此，这张卡最正确的用途不是“证明物理支路已经完成”，而是把第 2 章的物理线真正推进到一个工作状态：**我们现在已经有了能填数、能复现、能给支持也能给残差的第一张公开验证卡。**

### 2.3.6 首张失败登记卡：跨公开源直接拼一条原始 `K(\Pi)` 总律

如果物理线只收支持卡，不收失败卡，它很快就会滑回另一种老路：一看到多条公开线都出现 `K` 随 `\Pi` 塌缩，就急着把它们直接拼成一条跨对象、跨协议、跨单位的原始总律。当前最该先登记的失败，不是“窗口核已经被推翻”，而是：**把不同公开源未经整理就压成同一条绝对 `K(\Pi)` 式，本身就是错误动作。**

当前最小失败卡可以先写成：

```text
失效登记卡编号：PHY-FAIL-POOL-01
失败类型：跨公开源原始拼接失效
样本族：PulseBat / Battery Pulse Diagnostics / Galijatović / FZJ HPPC
表面目标：直接写出跨公开源通用 K(Π) = aΠ^{-η}
失效点：单位不统一、协议不统一、局部对象 z 不统一、η 跨源漂移
当前判定：原始绝对总律不得成立；只能先做同单位局部拟合或归一化后 pooled check
当前动作：保留局部窗口核；把跨源比较降级为归一化比较，而不是绝对量总拟合
```

这张失败卡当前已经能被公开表格直接支住：

| 公开源 | 可比量 | `\eta` 首轮估计 | `\Pi` 范围 |
| --- | --- | ---: | ---: |
| `PulseBat` | `K_{\mathrm{per\,A}}` | `0.9343` | `0.48s \rightarrow 80s` |
| `Battery Pulse Diagnostics` | `K_{\mathrm{per\,C}}` | `1.1896` | `8s \rightarrow 50s` |
| `Galijatović` | `K_{\mathrm{per\,C}}` | `1.1532` | `10s \rightarrow 910s` |
| `FZJ HPPC` | `K_{\mathrm{per\,A}}` | `1.5450` | `70.0s \rightarrow 610.0s` |

这组数已经足够把两个误用一起拦下来。

第一，**`\eta \approx 1` 目前只能当局部首轮近似，不能当跨源绝对常数。**  
从 `0.9343` 到 `1.5450`，跨源斜率漂移已经很明显；如果这时还把它们硬并成一个未分层的原始总律，那么所谓“统一”只是把协议差、对象差和单位差一起糊掉。

第二，**`K` 的量纲和归一方式不同，直接并表本身就不合法。**  
当前公开线里，有的更适合看 `K_{\mathrm{per\,A}}`，有的更适合看 `K_{\mathrm{per\,C}}`；再叠加不同电芯、不同方向、不同协议段，`z` 并没有被保持住。也就是说，这里失败的不是“局部塌缩趋势不存在”，而是“未经切片和归一化就想升格成总律”的写法不成立。

但这张失败卡也不是纯否定。它同时给出了一条更稳的出路：在把每条公开线先按各自最快可比 `\Pi` 点归一化后，当前公共池还能压出一条 `\eta \approx 0.9545`、`R^2 \approx 0.916` 的 pooled normalized loglog 检查。它说明：**跨源共同趋势可以检查，但必须先归一化，再比较；不能直接拿绝对量拼总式。**

因此，这张失败卡真正保护的不是某个漂亮公式，而是第 1 章冻结主线的边界：**物理线可以继续保留局部窗口核，也可以保留“归一化后存在共同塌缩趋势”的工作假设，但当前绝不能把原始跨源 `K(\Pi)` 总律写成已站住的结论。**

## 2.4 学习样本：验证概率有界窗口核

学习线当前要验证的，是另一个更窄的命题：

> 在固定局部历史状态后，概率有界窗口核是否能比单指数、幂律或现成经验模型更稳地给出保持边界和可行复习区间。

### 2.4.1 当前要收的量

学习样本至少要收：

1. 刷新周期 `\Pi`；
2. 局部刷新后概率上界 `p_0(z)`；
3. 局部有效衰减率 `\lambda(z)`；
4. 快周期软尺度 `\tau_f(z)`；
5. 硬下界 `\tau_h(z)`；
6. 最低可接受保持率 `p_{\mathrm{crit}}`。

局部切片至少要分到：

1. `history`；
2. `language pair`；
3. `lexeme / task family`；
4. 如有必要，再分课程结构、难度或题型。

### 2.4.2 当前要验的四件事

1. **`p_0(z)` 是否稳定可估**
   不能只说“看起来有上界”，而要看不同切片下的上界是否有一致识别方式。
2. **`\lambda(z)` 是否比全局衰减率更有解释力**
   若局部切片的 `\lambda(z)` 没有比粗分层更稳，就说明切片方式还不对。
3. **`\tau_f(z)` 是否真能改进快周期边界**
   若加入快周期项后只改善局部拟合，却不改善边界预测，则应降级其地位。
4. **窗口区间是否优于更简单模型**
   最关键的问题不是“这条式子能不能拟合”，而是它是否比单指数、幂律或 half-life regression 更稳地给出可行区间。

### 2.4.3 学习线的失败标记

1. `p_0(z)` 和 `\lambda(z)` 在同一切片下仍剧烈漂移；
2. 快周期端没有明显掉效，`\tau_f(z)` 进入后只带来参数冗余；
3. 左右边界无法和真实保持率阈值对上；
4. 分层后仍无法明显优于更简单模型；
5. 所谓“局部稳定”其实只是样本太少造成的偶然平滑。

### 2.4.4 首张实填验证卡：Duolingo `en->es` 的 fixed-history 对角线

学习线如果还只停在“以后要分层”，也还不够硬。当前最适合先落成一张实填卡的，是已经在第 1 章里压出全量结果的 `Duolingo en->es, session_seen=1` 这条公开线。它的好处不是“已经代表全部学习”，而是：局部切片清楚，样本量大，`p_0(z)`、`\lambda_{\mathrm{eff}}(z)` 和 `\Pi_{\max}` 都已经能直接从公开 CSV 反算出来。

当前可直接挂接的复现编号有两条：

1. `LRN-DUO-GRID-ENES-01`
   负责输出 `en->es`、`session_seen=1`、`history_seen \le 10` 的 fixed-history 网格；
2. `LRN-DUO-PIMAX-01`
   负责在给定保持阈值后，直接反算学习支路的局部 `\Pi_{\max}`。

若把它压成一张最小验证卡，当前更稳的写法可以直接记成：

```text
验证卡编号：LRN-DUO-GRID-ENES-01
样本类型：公开学习日志
局部切片：Duolingo / en->es / session_seen=1 / 对角线 fixed-history
比较轴：状态 = 1/1, 2/2, 3/3, …, 10/10
观测量：history_seen, history_correct, recall bins, obs
反算量：p0(z), λ_eff(z), Π_max(z | p_crit)
当前用途：验证局部有界衰减骨架、阈值敏感性、fixed-history 是否足够稳定
```

这张卡当前至少可以先填下面这组数：

| 状态 | obs | `p_0(z)` | `\lambda_{\mathrm{eff}}` (`day^-1`) | `R^2` | `\Pi_{\max}@0.90` (天) | `\Pi_{\max}@0.80` (天) |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `1/1` | `79,147` | `0.902383` | `0.000784` | `0.8378` | `3.37` | `153.61` |
| `2/2` | `117,807` | `0.912942` | `0.000756` | `0.6885` | `18.89` | `174.68` |
| `3/3` | `163,388` | `0.905758` | `0.000810` | `0.5248` | `7.87` | `153.28` |
| `8/8` | `41,491` | `0.923296` | `0.000796` | `0.5006` | `32.10` | `180.07` |
| `10/10` | `25,935` | `0.923060` | `0.000913` | `0.4331` | `27.71` | `156.72` |

这张卡当前最重要的，不是“又多了一组平台数据”，而是它已经把学习支路压出了三条首轮判断。

第一，**局部有界衰减骨架已经不再只是占位式。**  
在同一语言对、同一 `session_seen` 和同一 fixed-history 纪律下，`p_0(z)` 与 `\lambda_{\mathrm{eff}}(z)` 都已经能被稳定提取出来。这意味着学习支路当前至少已经有资格从“方法猜想”进入“局部可拟合对象”。

第二，**fixed-history 的主异质性目前更明显地落在 `\lambda_{\mathrm{eff}}(z)` 上，而不是 `p_0(z)` 上。**  
在上面这组对角线状态里，`p_0(z)` 大致落在 `0.902` 到 `0.923` 之间，而 `\lambda_{\mathrm{eff}}(z)` 落在 `7.56\times 10^{-4}` 到 `9.13\times 10^{-4}` 之间。它们都在变，但当前更像主导项的仍是衰减率，而不是刷新后天花板。

第三，**学习窗口对阈值极度敏感。**  
同一批局部切片，只把最低可接受保持率从 `0.80` 提到 `0.90`，可行周期就会从约 `154` 到 `180` 天，骤缩到约 `3` 到 `32` 天。这说明学习支路当前更该被理解为“局部阈值窗口问题”，而不是一个凭经验喊出的固定复习节奏。

但这张卡的价值还不止于支持。它也同时暴露出一个必须保留的边界：**对角线 fixed-history 还不能被偷换成“复习次数越多，遗忘率就必然单调下降”的简单定律。**  
在这张卡里，`1/1` 到 `10/10` 的 `\lambda_{\mathrm{eff}}` 并没有机械单调下降，`R^2` 也只在部分状态上进入较稳区。这说明：只按对角线状态收束，仍然不足以完成学习线的最终定稿。

### 2.4.5 这张实填卡当前支持什么，不支持什么

若把上面的数进一步压成可执行判断，那么这张卡当前支持的是：

1. 学习支路已经足以把 `p_0(z)`、`\lambda_{\mathrm{eff}}(z)` 和 `\Pi_{\max}` 从空变量压成可复现输出；
2. 局部有界衰减式已经能直接进入公开学习日志，而不必继续借物理支路硬套；
3. 阈值敏感性已经足够强，值得在后续验证中单独作为比较轴保留；
4. fixed-history 至少是一个必要切片，而不是可以省略的细节。

这张卡当前还**不能**支持的是：

1. 只靠 fixed-history 对角线就写出通用复习定律；
2. 把 `p_0(z)` 的窄分布直接抬高成认知系统统一上界；
3. 把 `\lambda_{\mathrm{eff}}(z)` 当成不受 `language pair`、`lexeme` 或课程结构影响的常数；
4. 仅凭这一条公开线，就宣称学习支路已经优于单指数、幂律或 half-life regression 的全部替代模型。

因此，这张卡在第 2 章里的正确角色，不是“学习线已经证明完毕”，而是：**我们现在已经有了一条能反算参数、能给窗口、也能暴露边界不足的公开学习验证卡。**

### 2.4.6 首张失败登记卡：`pt->en` 的高 history 对角线退化

学习线如果只收 `en->es` 这类较稳切片，也会迅速产生一种错觉：只要 fixed-history 切得够细，局部窗口就会自动越来越稳。当前最适合先登记的失败，不是“学习窗口不存在”，而是：**有些语言对在高 history 端会出现样本变薄、拟合塌缩和边界反跳，fixed-history 对角线本身并不能保证稳定。**

当前最小失败卡可以先写成：

```text
失效登记卡编号：LRN-FAIL-DUO-PTEN-01
失败类型：高 history 端退化 / 切片失稳
样本线：Duolingo / pt->en / session_seen = 1 / 对角线 fixed-history
表面目标：沿 1/1, 2/2, …, 10/10 平滑外推局部窗口
失效点：obs 下降、拟合缺值、λ_eff 跳变、Π_max 反跳
当前判定：fixed-history 对角线不能单独承担通用复习律
当前动作：补 pair × lexeme × state 交叉切片，并把高 history 端单列为脆弱区
```

这张失败卡当前至少能被下面这组数支住：

| 状态 | obs | `p_0(z)` | `\lambda_{\mathrm{eff}}` | `R^2` | `\Pi_{\max}@0.90` (天) |
| --- | ---: | ---: | ---: | ---: | ---: |
| `7/7` | `5,779` | `0.934458` | `0.000519` | `0.1744` | `72.39` |
| `8/8` | `4,194` | `0.939377` | `0.000650` | `0.5861` | `65.88` |
| `9/9` | `3,223` | — | — | — | — |
| `10/10` | `2,426` | `0.942145` | `0.001688` | `0.6858` | `27.11` |

这组数的价值非常大，因为它正好把“局部窗口已经稳了”的错觉掐断。

第一，**高 history 端的样本量下降会直接把局部拟合推向脆弱区。**  
从 `7/7` 的 `5,779` 条到 `10/10` 的 `2,426` 条，对角线状态已经明显变薄；到了 `9/9`，当前表里甚至直接出现了 `p_0(z)`、`\lambda_{\mathrm{eff}}(z)` 与 `R^2` 的空缺。这说明：fixed-history 不是切得越细越稳，切得过细也会把自己切到不可估。

第二，**即使没有完全缺值，边界也可能出现不该被硬解释成“真实认知跃迁”的跳变。**  
在这组状态里，`\lambda_{\mathrm{eff}}` 从 `7/7` 的 `0.000519` 跳到 `10/10` 的 `0.001688`，`\Pi_{\max}@0.90` 也从 `72.39` 天骤缩到 `27.11` 天。如果这时还强行沿对角线讲一个“复习次数越多，窗口越稳定”的单调故事，那就等于拿切片失稳替代结构判断。

第三，**这张失败卡打中的不是窗口模型本身，而是“只靠对角线 fixed-history 就足以完成学习线”的过强写法。**  
换句话说，当前更诚实的结论不是“学习窗口不成立”，而是“在某些语言对的高 history 端，pair、lexeme、task family 或样本厚度已经重新进入主导位置”。这也正是为什么第 2 章必须把失败卡和支持卡并列保留。

因此，这张卡真正保护的是学习线的边界：**fixed-history 仍然是必要切片，但它不是充分切片；高 history 端一旦开始退化，就必须补交叉分层，而不能继续拿对角线硬撑通用定律。**

### 2.4.7 一张桥接卡：大模型训练对学习支路有启发，但不能替代证据

到这里为止，学习线已经有了支持卡，也有了失败卡。此时最容易出现的另一种误写，是把“大模型如何训练”直接拿来当作学习窗口已经被证明的额外证据。当前更稳的做法，是把这类材料单独压成一张**桥接卡**：它可以帮助解释为什么学习支路值得保留、为什么某些训练动作会改写局部参数，但它**不能**替代公开样本、拟合与失败登记本身。

这张桥接卡当前最值得吸收的，不是“人脑等于 Transformer”这种过强类比，而是下面四条更稳的协议启发：

1. **高质量输入优于纯数量堆积；**
2. **主动预测、主动回忆和即时反馈，比被动重读更接近真实更新；**
3. **训练任务必须与输出目标对齐，而不是学了很多却不进入最终输出形式；**
4. **学习不是一次性完成，而需要沿着遗忘边界做周期刷新。**

若把这些启发压回当前学习支路，它们更适合被理解成：**不是在窗口核外再长一套心理学口号，而是通过协议改写局部状态 `z`，从而改写 `p_0(z)`、`\lambda_{\mathrm{eff}}(z)`、`\tau_h(z)`、`\tau_f(z)` 与高层输出区间。**

为避免这一步停在散文层，当前可先把“学习协议状态”统一记成

$$
\boxed{
z_{\mathrm{learn}}
=
\big(
z_{\mathrm{mat}},\
\phi_{\mathrm{in}},\
\phi_{\mathrm{pred}},\
\phi_{\mathrm{fb}},\
\phi_{\mathrm{align}},\
q_{\mathrm{in}},\
\ell_{\mathrm{out}}
\big),
}
$$

其中：

- `z_{\mathrm{mat}}`：材料难度、任务类型、对象家族等原始内容切片；
- `\phi_{\mathrm{in}}`：输入质量因子；
- `\phi_{\mathrm{pred}}`：主动预测 / 主动回忆强度；
- `\phi_{\mathrm{fb}}`：反馈可得性与反馈及时性；
- `\phi_{\mathrm{align}}`：学习任务与最终输出形式的一致度；
- `q_{\mathrm{in}}`：输入批次或学习材料进入的外部节律；
- `\ell_{\mathrm{out}}`：从底层掌握真正进入高层输出所需的组织、压缩、写作、讲解或答题准备延迟。

在这组状态变量下，当前更稳的写法不是先硬造一个新的认知总方程，而是先把协议效应压成局部参数的方向约束：

$$
\boxed{
\phi_{\mathrm{in}}\uparrow,\ 
\phi_{\mathrm{pred}}\uparrow,\ 
\phi_{\mathrm{fb}}\uparrow,\ 
\phi_{\mathrm{align}}\uparrow
\quad\Longrightarrow\quad
p_0(z_{\mathrm{learn}})\uparrow,\ 
\lambda_{\mathrm{eff}}(z_{\mathrm{learn}})\downarrow.
}
$$

这条方向式现在还不是实证定律，但它已经给第 2 章提供了一条很重要的解释纪律：  
**输入质量、主动回忆、反馈与输出对齐，并不是“窗口外”的学习技巧，而是会通过 `z` 切片进入窗口参数。**

一旦把这条桥接纪律接回第 4 章，当前就能得到一条对人类学习非常直接、而且比“多复习一点”更硬的高层输出式。  
若取学习退化支路，也就是当前公开样本更稳支持的

$$
p_{\mathrm{win},i}^{\mathrm{deg}}(\pi\mid z_i)
=
p_{0,i}(z_i)e^{-\lambda_i(z_i)\pi},
$$

并把一次高层输出任务看成跨过若干底层复习步的高层动作，则第 4 章已经给出的退化高层执行上界可以直接专化为：

$$
\boxed{
\Pi_{\mathrm{out},\max}^{\mathrm{learn}}(z)
=
\min\!\left\{
\ell_{\mathrm{out}}(z)
+
\frac{-\ln \rho_H}{\Lambda_H^{\mathrm{deg}}(z)},\
\ell_{\mathrm{out}}(z)
+
\min_i\left[
m_i(z)\,
\frac{\ln\!\big(p_{0,i}(z_i)/p_{\mathrm{crit}}\big)}{\lambda_i(z_i)}
\right]
\right\},
}
$$

其中

$$
\boxed{
\Lambda_H^{\mathrm{deg}}(z)
=
\sum_i w_i(z)\frac{\lambda_i(z_i)}{m_i(z)}.
}
$$

这条式子的意义很直接：

1. 第一项给出归一化健康阈值 `\rho_H` 下的软上界；
2. 第二项给出底层各条学习线都不能跌破 `p_{\mathrm{crit}}` 的硬上界；
3. 真正的高层输出周期，必须同时服从这两条上界里更紧的一条。

若再把输入批次节律接进来，则高层学习输出区间当前可以直接写成：

$$
\boxed{
\mathcal{I}_{\mathrm{out}}^{\mathrm{learn}}(z)
=
\left[
\max\!\Big\{
\max_i\!\big[m_i(z)\tau_{h,i}(z_i)+\ell_{\mathrm{out}}(z)\big],\
r^{-}(z)q_{\mathrm{in}}(z)
\Big\},
\ 
\min\!\Big\{
\Pi_{\mathrm{out},\max}^{\mathrm{learn}}(z),\
r^{+}(z)q_{\mathrm{in}}(z)
\Big\}
\right].
}
$$

这里：

- 左边界表示“太快也不行”：底层巩固时间和输出准备时间还没过，就不该进入高层输出；
- 右边界表示“太慢也不行”：底层保持、归一化健康度和输入批次都可能先把这次输出任务卡死。

因此，若把“大模型训练给人类学习的启发”压到当前主线里，最稳的吸收不是“人脑像 Transformer”，而是：

$$
\boxed{
\text{输入质量、主动预测、反馈与输出对齐}
\ \text{会通过局部状态 }z\text{ 改写学习窗口参数，}
\text{从而改写高层输出区间。}
}
$$

这也正好把前面那类外部回答放回它应有的位置：  
它的角色不是**学习支路已经被证明**，而是**学习支路如何被协议动作主动调参**。  
换句话说，它更适合充当第 2 章的**桥接卡**，而不是新的**证据卡**。

若后续需要把“大模型训练里的公开算法公式”继续压成更细的差异—耗散—周期语言，则更合适的承接位置不是本章，而是 `AI协作卷` 的对应专题章。第 2 章在这里先只保留桥接职责，不再继续向训练算法细节扩写，以免样本账本和方法桥接重新混写。

## 2.5 AI 协作样本：先立验证卡，再决定是否值得写核

AI 协作线是当前最需要补的，但也最不适合一上来就假装自己已经有完整闭式。

这里更稳的入口不是先写一条漂亮公式，而是先写一个最小验证判据：

$$
Q(\Pi\mid z)\ge Q_{\mathrm{crit}},
\qquad
\Pi\in \mathcal{I}_{\mathrm{AI}}(z).
$$

这里的 `Q` 暂时不预设为单一物理量，而只代表一类可记录的协作质量保持量。第 2 章的任务，是先把 `Q` 落成观测卡，而不是立刻给出最终窗口核。

### 2.5.1 当前优先记录的质量量

AI 协作样本至少可以从下面六类量里先选：

1. 回写采纳率；
2. 评论解决率；
3. 审核通过率；
4. 主线冲突率的反向指标；
5. 撤回 / 重做率；
6. 问题积压时间。

### 2.5.2 当前优先记录的周期量

1. AI 汇总周期；
2. 人类审核周期；
3. 回写周期；
4. 版本冻结周期；
5. 问题积压清空周期。

### 2.5.3 AI 线当前的最小实验

首轮实验不需要很大，但必须能比较不同周期：

1. 固定一批评论或问题；
2. 设定两到三种回写周期，例如每日、三日、每周；
3. 记录每种周期下的审核耗时、采纳率、冲突数和撤回率；
4. 比较不同周期下质量谷底是否出现明显窗口。

只有当这一步显示出稳定窗口趋势，AI 协作支路才有资格在后续章节里继续压成完整窗口核。

### 2.5.4 AI 线的失败标记

1. 不同周期下质量量没有明显差异；
2. 指标主要受人员差异而非周期差异主导；
3. 回写质量没有谷底结构，只表现为单调噪声；
4. 不同批次任务之间无法形成可比切片。

若这些失败标记持续存在，第 2 章更稳的结论就不是“AI 协作不服从窗口”，而是“当前指标与切片方式还不足以支持窗口检验”。

### 2.5.5 首张可执行验证卡：评论归并到正式回写

AI 协作线目前还没有像物理线、学习线那样的公开数表，所以这里更稳的推进不是硬造一个漂亮曲线，而是先把“首轮验证卡”压到真正可以执行的粒度。当前仓库里已经有一套足够好的协作词表：`评论生长卡`、`回写修改任务单`、`版本更新单`、`反馈回写表`。因此，第 2 章更适合先直接挂接这条现成流程，而不是另外发明一套新术语。

若把这条线压成最小验证卡，当前可以先写成：

```text
验证卡编号：AI-WRITEBACK-01
样本类型：评论归并 -> 回写 -> 审核 -> 版本留痕
局部切片：同一批评论 / 同一章节族 / 同一冻结规则 / 同一审核人力条件
比较轴：AI 汇总周期 = 每日 / 三日 / 每周
观测量：评论数、进入生长卡数、进入任务单数、正式回写数、版本单数
质量量：采纳率、主线冲突率、撤回率、积压时间、评论解决率
当前用途：验证回写周期是否存在过快失真与过慢积压的双端问题
```

这张卡的关键，不在于先把 `Q(\Pi\mid z)` 压成单一数，而在于先把质量拆成可记录的几个分量：

1. `A(\Pi\mid z)`：回写采纳率；
2. `C(\Pi\mid z)`：主线冲突率；
3. `R(\Pi\mid z)`：撤回 / 重做率；
4. `B(\Pi\mid z)`：积压时间或积压量；
5. `S(\Pi\mid z)`：评论解决率。

当前更稳的首轮可行判据不是强行写

$$
Q=A(1-C)(1-R)
$$

这种新总式，而是先写成多条件门槛：

$$
\boxed{
A(\Pi\mid z)\ge A_{\mathrm{crit}},
\quad
C(\Pi\mid z)\le C_{\mathrm{crit}},
\quad
R(\Pi\mid z)\le R_{\mathrm{crit}},
\quad
B(\Pi\mid z)\le B_{\mathrm{crit}}.
}
$$

只要这四类门槛开始稳定分出“过快 / 可行 / 过慢”三段，AI 线就已经进入了窗口检验，而不必急着宣告自己已经有完整闭式。

### 2.5.6 这张 AI 卡怎么真正执行

当前最小实验可以直接压成四步：

1. 固定一批评论
   最好来自同一章节族，同一轮读者反馈，不把主题差异过大的评论混在一起；
2. 固定冻结规则
   明确这轮是局部修补、补桥、降强，还是允许小节级改写；
3. 比较三种回写周期
   例如每日、三日、每周；
4. 每轮都记录五组量
   评论进入率、回写采纳率、主线冲突率、撤回率、积压时间。

若把它和 AI 协作卷已有模板接起来，当前更稳的记录顺序可以写成：

1. 评论先进 `评论生长卡`；
2. 真的准备动正文的，进 `回写修改任务单`；
3. 改动结构后，补 `版本更新单`；
4. 读者侧接收结果，再进 `反馈回写表`。

这样一来，AI 线就不再只是“以后也许能测”，而是已经有了一条最小可执行验证链。

### 2.5.7 这张 AI 卡当前支持什么，不支持什么

这张卡当前支持的是：

1. AI 协作线已经可以先不靠新公式，而靠现有工作流对象进入样本验证；
2. “过快回写”与“过慢积压”都已经能被翻译成可记录的质量门槛；
3. 第 2 章现在至少能为 AI 线留下一个真实可跑的实验接口，而不是只留口号。

这张卡当前还**不能**支持的是：

1. AI 协作已经存在单一稳定窗口核；
2. 采纳率、冲突率和积压时间已经能自然压成一个统一标量；
3. 只凭一轮实验就能宣布 AI 回写周期的最佳点已经找到。

因此，AI 线在本章更诚实的位置是：**它已经有了第一张能开工的验证卡，但还没有资格跳过实验，直接进入闭式。**

### 2.5.8 首张失败登记卡：跨章节混池时，回写周期比较无效

AI 线当前没有公共数表，所以它最先需要登记的失败，往往不是“公式没拟合上”，而是“实验一开始就没切对片”。也就是说，AI 协作线的第一个失败卡，最好不是理论失败卡，而是**协议失配 / 切片失效卡**。

当前最小失败卡可以先写成：

```text
失效登记卡编号：AI-FAIL-MIXPOOL-01
失败类型：切片失效 / 协议失配
样本线：评论归并 -> 回写 -> 审核 -> 版本留痕
表面目标：直接比较每日 / 三日 / 每周回写周期
失效点：同一批次混入不同章节族、不同冻结规则、不同审核负担
当前判定：该批记录不得进入 CMP-AI-WRITE-01
当前动作：先拆回评论生长卡、回写修改任务单、反馈回写表，再按章节族与冻结规则重切
```

这张卡现在还没有公共数表，但它仍然应该先被写进第二章，因为 AI 线最容易发生的首轮失败，本来就不是“窗口不存在”，而是下面这三种更基础的失效：

1. **章节族混池**
   把定义章、推演章和导读章的评论混成一批后，采纳率与冲突率首先反映的是章节职责差异，而不是周期差异；
2. **冻结规则混池**
   有的批次只允许措辞修补，有的批次允许小节级重写；一旦把它们并表，撤回率与审核耗时就会被改动权限主导；
3. **审核负担混池**
   人力宽松与人力紧张的两轮批次，即使回写周期一样，积压时间和清空率也不可直接比较。

这意味着：**AI 线的第一个失败判断，常常不是“没有窗口”，而是“没有形成可比切片”。**  
如果这一关不过，后面的每日、三日、每周比较几乎注定只能得到噪声；而这种噪声若被误写成“AI 协作没有周期窗口”，同样是一种过度结论。

因此，这张失败卡在本章里的职责非常明确：它提醒我们，AI 线的下一步首先不是去找闭式，而是去做实验卫生。当前仓库里已经有现成对象可挂：`评论生长卡` 负责先分流碎片，`回写修改任务单` 负责锁定改动边界，`反馈回写表` 负责记录接收结果，`版本更新单` 负责留下结构痕迹。只要这几步没有先压平，AI 线的周期比较就不应被记作有效样本。

## 2.6 模型对照：不只证明能拟合，还要证明更值得用

本章至少要做三类对照。

### 2.6.1 物理线对照

1. 无快周期惩罚：
   $$
   \frac{K_1}{e^{\lambda\Pi}-1}
   $$
2. 完整窗口：
   $$
   \frac{K_1e^{-\tau_f/\Pi}}{e^{\lambda\Pi}-1}
   $$
3. 幂律经验式：
   $$
   a\Pi^{-b}
   $$
4. 单指数经验式：
   $$
   ae^{-b\Pi}
   $$

### 2.6.2 学习线对照

1. 单指数：
   $$
   p_0e^{-\lambda\Pi}
   $$
2. 完整窗口：
   $$
   p_0e^{-\lambda\Pi-\tau_f/\Pi}
   $$
3. 幂律遗忘：
   $$
   p_0(1+a\Pi)^{-b}
   $$
4. logit 线性；
5. half-life regression 原模型。

### 2.6.3 AI 协作线对照

1. 固定快周期回写；
2. 固定慢周期回写；
3. 无冻结、只滚动回写；
4. 带冻结与审核节律的窗口式回写。

本章不要求这些对照一次性全部完成，但要求至少为每条样本线建立一组可重复比较的基线。

### 2.6.4 比较不能只看残差，还要看四类共同指标

如果不先统一比较指标，“模型对照”最后很容易退化成各自挑对自己有利的结果。当前更稳的做法，是先把三条样本线的共同比较轴压成四类：

1. **切片内拟合**
   在同一局部切片里，拟合误差是否足够低；
2. **边界命中**
   左右边界、失稳点或阈值穿越点是否能被正确判到；
3. **切片外稳定性**
   换一个相邻切片、相邻协议或相邻状态后，参数和判断是否还能保持同量级；
4. **复杂度成本**
   新模型引入的新参数，是否真的换来了更稳的边界判断，而不是只换来了局部过拟合。

若把它压成最小比较表，当前可以统一写成：

| 比较维度 | 物理线 | 学习线 | AI 协作线 |
| --- | --- | --- | --- |
| 切片内拟合 | `\Delta_{\mathrm{proxy}}(\Pi)` 的误差 | 保持率或谷底概率误差 | 质量量门槛命中率 |
| 边界命中 | `\Pi_{\min}, \Pi_{\max}` 与失稳区是否对上 | `\Pi_{\max}` 与阈值保持率是否对上 | 过快失真 / 过慢积压是否被判到 |
| 切片外稳定性 | 换协议、换电芯、换 SOC 后是否同量级 | 换 state / pair / lexeme 后是否仍站住 | 换章节族、换审核负担后是否仍可用 |
| 复杂度成本 | 新参数是否真改善边界预测 | 新参数是否真改善窗口判断 | 冻结与审核规则是否值得额外流程成本 |

这样一来，后面的对照就不再只是“谁看起来更像曲线”，而是开始进入：**谁更能稳地判边界、谁更能跨切片复用、谁的额外复杂度更有价值。**

### 2.6.5 当前已经能做的比较，与当前还不能做的比较

把第 1 章和本章现有样本线合在一起看，当前并不是每一类对照都已经同样成熟。更稳的判断应分三档写。

#### 第一档：现在就能直接做的比较

1. **物理线：无快周期惩罚 vs 完整窗口**
   这一条当前已经有现成脚本接口：`fit_cycle_model.py` 可先给出无快周期骨架，`compute_physical_window_bounds.py` 与 `compute_public_physical_window_grid.py` 已能给出完整窗口的边界与近峰收缩检查。
2. **物理线：完整窗口 vs 简单经验式**
   至少在 `Battery Pulse Diagnostics`、`PulseBat`、`Galijatović`、`FZJ HPPC` 这几条公开线里，已经能先比较“谁更能解释 `K_{\mathrm{proxy}}` 随 `\Pi` 的塌缩”和“谁更能给出可执行边界”。
3. **学习线：单指数局部式 vs 阈值窗口反算**
   虽然这两者还属于同一骨架的不同输出层，但当前已经能直接检查：只靠 `p_0(z), \lambda_{\mathrm{eff}}(z)` 反算出的 `\Pi_{\max}`，是否和设定阈值下的可行区间一致。
4. **AI 线：固定快周期 / 固定慢周期 / 滚动回写 / 带冻结回写**
   这条线虽然还没有闭式，但工作流对照已经可以先开始做。

#### 第二档：已经有比较对象，但还不能算“完成比较”

1. **学习线：单指数 vs 完整窗口**
   目前 `p_0(z)e^{-\lambda\Pi}` 这条局部式已经能稳拟合，但含 `\tau_f` 的完整窗口还没有在同一批切片上系统估计出来，所以这组比较当前还只能算“接口已经有了”，不能算结果已经站住。
2. **学习线：单指数 / 完整窗口 vs 幂律 / HLR**
   这组对照是必须做的，但当前仓库里已经稳定产出的，主要还是单指数局部拟合与 `\Pi_{\max}` 反算；要把幂律和 half-life regression 放进同一张比较表，还需要先统一切片、统一指标和统一输出格式。

#### 第三档：当前还不能假装已经能做的比较

1. **AI 线：多指标直接压成单一 `Q(\Pi\mid z)` 闭式**
   现在还不该跳这一步。
2. **跨物理、学习、AI 三线直接拼成统一优胜模型**
   当前还没有这个条件。
3. **只凭一个样本族就宣布某条核“全面优于全部替代模型”**
   当前也还没有这个条件。

因此，本章在模型对照上的更稳结论应写成：**物理线现在已经进入正式对照期，学习线已经进入局部对照期，AI 线则刚进入工作流对照期。**

因此，这张卡现在给出的最稳判断不是“完整窗口已经彻底赢了”，而是：**在物理线首轮公开比较里，`τ_f` 已经通过边界功能证明了自己的进入必要性；无快周期骨架应保留为退化参考式，而不应继续冒充完成式。**

若只停在 `p_0(z)e^{-\lambda\Pi}`，我们得到的是局部拟合对象；一旦引入外置阈值 `p_{\mathrm{crit}}`，同一组参数立刻能被翻译成“什么周期还可接受、什么周期已经不可接受”的执行判断。并且，这个输出层带来的不是小修小补，而是数量级变化：保持率底线从 `0.80` 提到 `0.90`，可行周期就会从约半年量级骤缩到数天到数十天量级。

因此，这张卡当前给出的最稳结论也应写得克制：**阈值窗口反算已经证明自己是学习线里必要的执行层，但它还没有证明自己已经打败幂律遗忘、logit 线性或 half-life regression。** 这一步要等同切片、同指标、同输出格式的正式对照表补齐后，才能继续往前走。

也正因为三条线所处的成熟度不同，本章此刻最需要的不是强行统一胜负，而是把它们放回各自真实的位置：物理线继续补跨样本复核，学习线继续补替代模型同表比较，AI 线继续先做最小实验，而不是提前宣判。

### 2.6.8 对照之后，本章先把公式分层冻结

如果模型对照做到这里，还不把“哪些式子已经够资格升格、哪些还只能留在账本里”写死，那么下一轮很容易再次回到老问题：一边拿已站住的式子当占位，一边又把尚未站住的式子提前写成主干。为了防止这种口径漂移，第 2 章在回到后续推导前，应先冻结一张**公式分层卡**。

#### 第一层：现在就允许回写进主干的式子

这一层的式子，当前已经不只是“想法对”，而是已经有局部切片、公开样本接口和失败登记来支撑它们继续留在主线里。

1. **统一判据**

   $$
   \boxed{
   \Delta_{\mathrm{req}}(z)
   \le
   \Delta_{\mathrm{lim}}(\Pi\mid z),
   \qquad
   \Pi\in\mathcal{I}(z).
   }
   $$

   它当前统一的不是所有对象共享同一原始动力学核，而是共享同一套稳态判定程序。

2. **连续幅值最小骨架**

   $$
   \boxed{
   \Delta_{\mathrm{lim}}^{\mathrm{exact}}(\Pi)
   =
   \frac{K\Pi}{e^{\lambda\Pi}-1}.
   }
   $$

   这条式子当前可以作为模型内定理保留，但必须连同假设一起保留：周期补给、指数耗散、谷底判据成立。

3. **学习局部有界衰减式**

   $$
   \boxed{
   p_{\mathrm{floor}}(\Pi\mid z)
   \approx
   p_0(z)e^{-\lambda_{\mathrm{eff}}(z)\Pi}.
   }
   $$

   这条式子当前已经能进入学习支路主干，但仍只能写成局部日志切片上的工作式，而不是认知系统的总定律。

4. **学习执行边界反算式**

   $$
   \boxed{
   \Pi_{\max}^{\mathrm{learn}}(z)
   =
   \frac{\ln p_0(z)-\ln p_{\mathrm{crit}}}{\lambda_{\mathrm{eff}}(z)}.
   }
   $$

   这条式子的重要性在于，它把学习线第一次稳定地推进到了“能从拟合对象翻译成执行边界”的层面。

#### 第二层：当前允许保留为工程展开的式子

这一层的式子已经值得继续保留，但它们当前的地位仍是**条件成立时可用的工程展开**，还不能直接冒充主干定律。

1. **物理完整窗口核**

   $$
   \Delta_{\mathrm{win}}(\Pi\mid z)
   =
   \frac{K_1(z)e^{-\tau_f(z)/\Pi}}{e^{\lambda(z)\Pi}-1}.
   $$

   只有在短周期端确实观察到掉效、且 `\tau_f(z)` 能被正值识别时，它才继续高于退化骨架。

2. **学习完整窗口核**

   $$
   p_{\mathrm{win}}(\Pi\mid z)
   =
   p_0(z)e^{-\lambda(z)\Pi-\tau_f(z)/\Pi}.
   $$

   它当前是完整窗口候选式，而不是已经普遍站住的学习母式。

3. **局部幂律塌缩式**

   $$
   K(\Pi,z)
   =
   K_0(z)\left(\frac{\Pi}{\Pi_0}\right)^{-\eta(z)}.
   $$

   它当前可以保留为物理公开样本上的局部工程近似，但不能提前写成跨对象、跨协议的原始总律。

#### 第三层：当前只能保留为强假说或工作框架的式子

这一层的式子不是没有价值，而是它们的价值目前主要体现在组织研究、压出切片和设计实验，还不在于已经被样本正式站住。

1. **经验中心 `\eta\approx 1`**

   这只能写成当前公开物理样本上的经验中心，不能写成理论常数。

2. **高层投影与工作点公式**

   $$
   \mathcal{I}_H,\ \chi_H,\ g_H,\ \Pi_{H,\mathrm{work}},\ \theta_H^\star
   $$

   这一组式子当前更适合被理解为高层周期约束的工作框架，还缺稳定识别规则与跨样本复核。

3. **学习高层输出专化式**

   $$
   \Pi_{\mathrm{out},\max}^{\mathrm{learn}}(z),
   \qquad
   \mathcal{I}_{\mathrm{out}}^{\mathrm{learn}}(z)
   $$

   它们当前更适合保留为桥接卡中的高层专化结果，用来说明学习协议动作如何改写输出区间，而不是已经完成验证的主干公式。

4. **AI 单标量窗口核**

   $$
   Q_{\mathrm{win}}(\Pi\mid z)
   =
   Q_0(z)e^{-\lambda_Q(z)\Pi-\tau_v(z)/\Pi}
   $$

   这一条当前仍只是第三支路的候选核；在公共日志和首轮协议对照出来前，不应写成 AI 线主式。

#### 第四层：当前只能保留为实验判据，不能伪装成公式进展

AI 线当前最有价值的，并不是再造一个漂亮总式，而是把验证门槛先压成协议可执行的多条件判据：

$$
\boxed{
A(\Pi\mid z)\ge A_{\mathrm{crit}},
\quad
C(\Pi\mid z)\le C_{\mathrm{crit}},
\quad
R(\Pi\mid z)\le R_{\mathrm{crit}},
\quad
B(\Pi\mid z)\le B_{\mathrm{crit}}.
}
$$

这组判据当前推进的是实验卫生和工作流比较，不是闭式胜利。它们的职责，是先把“过快失真 / 过慢积压 / 切片失效”压成可记录事件。

把这四层写死以后，第 2 章对“公式进度”的推进就可以被更准确地理解为：

> **本章已经把最小骨架、执行边界和若干条件展开分开冻结；从现在起，公式能否继续升格，不再由它看起来多漂亮决定，而由它是否经得住切片、对照和失败登记决定。**

### 2.6.9 从“能算”到“能信”：把窗口公式升级成可信判定器

到这里还必须再把目标收紧一次。若后面说“这套公式开始值得信”，这里的“信”当前不能直接理解成“像相对论那样成熟可信”，而应先收缩为一层更诚实的目标：

$$
\boxed{
\text{工程可信}
=
\text{可复现}
+
\text{有不确定性}
+
\text{能对照}
+
\text{能失败}
+
\text{能给保守决策}.
}
$$

也就是说，本章下一步不该继续追求“更漂亮的总公式”，而应把现有窗口式升级成一套**可信判定器**：

$$
\boxed{
\text{公式}
+
\text{参数估计}
+
\text{置信区间}
+
\text{对照模型}
+
\text{失败规则}
+
\text{保守工作周期}.
}
$$

这里的分层必须和第 11 章保持一致：第 11 章保留的是**理论上可以直接信任的公式结构**，而本章负责的是把这些结构连同参数估计、不确定性、对照与失败规则，升级成**工程可信的决策输出**。两者都重要，但不能再混成同一层。

#### 先改写输出格式：参数不再只给点估计

前面学习支路已经站住的最小工作式是

$$
p_{\mathrm{floor}}(\Pi\mid z)
\approx
p_0(z)e^{-\lambda(z)\Pi}.
$$

但若只写成点估计，它还只能算“能算”，不能算“能信”。更稳的可信版本必须把参数写成估计量而不是单点：

$$
\boxed{
p_0(z)\sim \widehat p_0 \pm \sigma_{p_0},
\qquad
\lambda(z)\sim \widehat\lambda \pm \sigma_\lambda.
}
$$

于是，

$$
\Pi_{\max}(z)
=
\frac{\ln p_0(z)-\ln p_{\mathrm{crit}}}{\lambda(z)}
$$

也不应再被写成单点，而应写成分布或区间：

$$
\boxed{
\Pi_{\max}(z)
\in
\left[
\Pi_{\max}^{\mathrm{low}}(z),\
\Pi_{\max}^{\mathrm{high}}(z)
\right].
}
$$

这里当前不要求本章立刻给出每条样本线的 bootstrap 数值，但必须先把规则写死：**后续所有“可直接决策”的周期输出，都应来自 bootstrap、交叉验证或其他稳定重采样程序，而不能只拿单次拟合值直接排程。**

#### 学习支路可信公式卡 v1.0

若沿单侧学习模型推进，当前更稳的可信版本可先固定为：

$$
\boxed{
p(\Pi\mid z)
=
p_0(z)e^{-\lambda(z)\Pi}
+
\epsilon,
}
$$

其中 `\epsilon` 表示观测噪声、切片内未解释波动或协议残差。

对应的阈值周期仍是：

$$
\boxed{
\Pi_{\max}(z)
=
\frac{\ln p_0(z)-\ln p_{\mathrm{crit}}}{\lambda(z)},
\qquad
0<p_{\mathrm{crit}}<p_0(z)\le 1.
}
$$

但这一版必须连同一个新的失败条件一起输出：

$$
\boxed{
p_{\mathrm{crit}}\ge p_0(z)
\Rightarrow
\mathcal{I}(z)=\varnothing.
}
$$

也就是说，如果刷新后的局部上限本身都达不到目标阈值，那么当前该做的不是缩短周期，而是提高学习质量、改协议、换切片或直接降目标。

真正用于决策的工作周期也不再直接取 `\Pi_{\max}`，而应取保守分位数再乘安全系数：

$$
\boxed{
\Pi_{\mathrm{work}}(z)
=
s\cdot Q_{0.05}\!\bigl(\Pi_{\max}(z)\bigr),
\qquad
0<s<1.
}
$$

这里：

- `s\approx 0.7`：常规保守；
- `s\approx 0.5`：高风险或高要求任务；
- `s\approx 0.9`：低风险、低代价任务。

这条改写的意义很大，因为它把“最大可行周期”与“实际工作周期”第一次明确分开。前者属于模型边界，后者属于保守决策。

#### 完整窗口支路也必须升级成保守窗口

若后续某些学习切片确实识别到快周期掉效，则完整窗口支路可写成：

$$
\boxed{
p_{\mathrm{win}}(\Pi\mid z)
=
p_0(z)\exp\!\left[
-\lambda(z)\Pi
-\frac{\tau_f(z)}{\Pi}
\right].
}
$$

这时最优周期与双边界仍可沿原式推进：

$$
\boxed{
\Pi_\ast(z)=\sqrt{\frac{\tau_f(z)}{\lambda(z)}},
}
$$

$$
\boxed{
\Pi_{L,R}(z)
=
\frac{
L(z)\mp\sqrt{L(z)^2-4\lambda(z)\tau_f(z)}
}{
2\lambda(z)
},
\qquad
L(z)=\ln\frac{p_0(z)}{p_{\mathrm{crit}}}.
}
$$

但可信版本当前更关键的，不是再多解释一次双边界，而是把决策窗口改写为保守交集：

$$
\boxed{
\mathcal{I}_{\mathrm{safe}}(z)
=
\left[
Q_{0.95}\!\bigl(\Pi_L(z)\bigr),\
Q_{0.05}\!\bigl(\Pi_R(z)\bigr)
\right].
}
$$

解释很简单：

- 左边界要取偏大的保守值；
- 右边界要取偏小的保守值。

因此，完整窗口当前最该增加的一条失败规则是：

$$
\boxed{
Q_{0.95}\!\bigl(\Pi_L(z)\bigr)
>
Q_{0.05}\!\bigl(\Pi_R(z)\bigr)
\Rightarrow
\text{当前数据下没有可信可行窗口。}
}
$$

这样一来，完整窗口就不再只是“看起来很像有一段双边区间”，而是第一次具备了直接宣布“当前窗口不可信”的出口。

#### 物理支路可信公式卡 v1.0

连续幅值支路当前最稳的可信改写，仍应先从退化骨架走，而不是一上来把所有项都压满。若取当前工程上最容易使用的 `\eta\approx 1` 退化式，则有

$$
\boxed{
\Delta_{\mathrm{lim}}^{(1)}(\Pi\mid z)
=
\frac{K_1(z)}{e^{\lambda(z)\Pi}-1}.
}
$$

给定临界差异 `\Delta_{\mathrm{crit}}` 后，对应的最大周期可反解为：

$$
\boxed{
\Pi_{\max}^{(1)}(z)
=
\frac{1}{\lambda(z)}
\ln\!\left(
1+\frac{K_1(z)}{\Delta_{\mathrm{crit}}}
\right).
}
$$

但在可信版里，真正拿来排程的也不应是这个点估计，而应是保守下界：

$$
\boxed{
\Pi_{\mathrm{safe}}^{\mathrm{phys}}(z)
=
Q_{0.05}\!\left[
\frac{1}{\lambda(z)}
\ln\!\left(
1+\frac{K_1(z)}{\Delta_{\mathrm{crit}}}
\right)
\right].
}
$$

若物理完整窗口核后续也能稳定识别到 `\tau_f(z)`，则其可信输出同样应统一改写为保守窗口：

$$
\boxed{
\mathcal{I}_{\mathrm{safe}}^{\mathrm{phys}}(z)
=
\left[
Q_{0.95}\!\bigl(\Pi_L(z)\bigr),\
Q_{0.05}\!\bigl(\Pi_R(z)\bigr)
\right].
}
$$

这一步的价值在于：它把物理线也从“给一个理论边界点”推进成“给一个可保守执行的周期界”。

#### 可信度不再只靠一句话，而应给一个等级

如果后面真的要让某条窗口式进入“可直接信任”的状态，本章还必须要求它同时输出一项可信度评分。当前最小可行写法可以先冻结成：

$$
\boxed{
R_{\mathrm{trust}}(z)
=
R_{\mathrm{fit}}
\cdot
R_{\mathrm{data}}
\cdot
R_{\mathrm{stable}}
\cdot
R_{\mathrm{compare}}
\cdot
R_{\mathrm{replicate}}.
}
$$

其中五项分别对应：

1. **拟合质量**：切片内误差是否足够低；
2. **数据量质量**：有效样本厚度是否足够支撑该切片；
3. **参数稳定性**：bootstrap 或重采样后，`p_0,\lambda,\tau_f,K_1` 是否漂移过大；
4. **对照优势**：当前模型是否真的优于更简单基线；
5. **复现程度**：换相邻切片、相邻协议或下一批样本后，是否还能站住。

这一版不要求现在就固定每一项的唯一打分公式，但要求后续所有“可信输出”都至少包含这五项职责。若要先给一版工程等级，可以暂定：

| `R_{\mathrm{trust}}` | 等级 | 当前使用方式 |
| ---: | --- | --- |
| `>0.8` | A | 可直接排程 |
| `0.6-0.8` | B | 可用，但需保守 |
| `0.4-0.6` | C | 只作参考 |
| `<0.4` | D | 不建议决策 |

这样做的意义，不是制造一个新分数崇拜，而是强制每次输出都回答一句：**这次给出的周期，到底是可直接排程，还是只够参考。**

#### 可信判定流程也要先写死

若把上面的式子和规则压成一张最小流程卡，当前更稳的执行顺序应是：

```text
输入数据
  ↓
固定局部切片 z
  ↓
拟合候选模型
  ↓
与简单模型对照
  ↓
bootstrap / 交叉验证
  ↓
计算 Π_max 或 [Π_L, Π_R]
  ↓
取保守分位数
  ↓
输出可信度等级
  ↓
若连续穿阈，则报警、降级或重拟
```

把它再压成一条门槛式，就是：

$$
\boxed{
\text{可直接信任}
\Longleftrightarrow
R_{\mathrm{trust}}(z)\ge R_0
\quad\text{且}\quad
\mathcal{I}_{\mathrm{safe}}(z)\neq\varnothing.
}
$$

若这两个条件任何一个不满足，则当前输出只能降级为：

$$
\boxed{
\text{可参考，或必须重切、重拟、重做对照。}
}
$$

这也正好把“相信它”这件事压回它真正该站的位置：可信，不再靠宣告，而靠程序。

#### 本章从现在起应如何改写“结果输出”

因此，今后本章以及后续样本线若要输出一个周期结果，最稳的格式不应再写成“最大周期约多少天”，而应统一改成下面这个模板：

```text
局部切片 z：
模型：
参数估计：
- p0 / lambda / tau_f / K1 = ...
不确定性：
- 置信区间 / bootstrap 分位数 = ...
阈值：
- p_crit 或 Delta_crit = ...
周期结果：
- Π_max 中位数 = ...
- 保守窗口 / 5% 下界 = ...
推荐工作周期：
- Π_work = ...
可信度：
- R_trust = ...
- 等级 = A / B / C / D
失败条件：
- 若出现……则报警、降级或重拟
```

如果把这一节再压到最短，它真正推进的不是一个新母式，而是一条新纪律：

> **从这一节开始，窗口公式若想进入“可信”状态，就必须同时带上不确定性、对照结果、失败出口与保守决策；只给单点估计的公式，今后都只能算“能算”，不能算“能信”。**

### 2.6.10 首张可信输出卡：Duolingo `en->es` 的 `8/8` 状态

1. 当前只完成了切片内 bootstrap，还没有完成 held-out 预测检验；

4. 因此，这张卡当前更准确的位置仍然是：

$$
\boxed{
\text{一张已经足以支持保守排程的 B/C 级工程可信卡，}
\text{而不是一张已可直接封口的成熟定律卡。}
}
$$

如果把这张卡压成一句话，它真正完成的不是“证明学习窗口定律已经成熟”，而是：

> **本章第一次把一个局部学习切片从点估计推进成了保守窗口与工作周期都可直接输出的可信样本卡。**

## 2.7 什么时候才允许回到公式升级

本章的一个重要职责，是替后续章节守住入口。上一节既然已经把“可升格 / 只可保留 / 暂不许冒充进展”的分层写死，这一节又已经把“能算”与“能信”的距离压成了可信判定器，后面若要继续新增公式项，就必须由本章留下的残差、失败记录和可信输出缺口来触发，而不能由“想到一个因素”直接触发。

更稳的升级条件是：

1. 现有窗口核在多组样本上出现同方向残差；
2. 残差能被某个明确遗漏机制解释；
3. 该机制能落成可观测量、可反算量或稳定代理量；
4. 新项能改善边界预测或失败预警，而不只是压低训练误差；
5. 至少有一个失败样本支持新增项的必要性。

换句话说，本章不是后续推导的过渡桥，而是后续推导的门卫。
