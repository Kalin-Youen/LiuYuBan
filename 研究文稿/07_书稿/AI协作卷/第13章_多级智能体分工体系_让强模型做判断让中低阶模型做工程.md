# 第13章 多级智能体分工体系：让强模型做判断，让中低阶模型做工程

## 13.1 为什么必须建立多级分工体系

本项目已经不适合继续采用“一个模型处理全部任务”的方式。原因很简单：项目材料越来越多，任务类型越来越复杂，如果所有任务都交给最强模型，会浪费高阶判断资源；如果把复杂判断交给较弱模型，又容易造成主线偏移、过度扩写、错误归类和低质量回写。

因此，需要建立一套多级智能体分工体系：

> **让最聪明的智能体负责判断、架构、边界、路线和验收；让中阶智能体负责执行、整理、初稿、批量处理和初步评级；让低阶智能体负责机械转换、格式清洗、索引生成和重复性检查。**

这不是为了制造等级感，而是为了提高系统效率。不同模型的能力不同，适合的任务也不同。真正高效的协作，不是让所有模型都“自由发挥”，而是把任务拆成不同层级，并为每一级设置清晰边界、交付格式和复核机制。

本章的目标是形成一套可以直接执行的协议：

```text
任务分级
→ 模型分级
→ 交付标准
→ 初步评级
→ 上级复核
→ 回写或退回
```

## 13.2 总原则：聪明模型做不可逆判断，普通模型做可复核工程

多级分工的第一原则是：

> **越不可逆、越影响主线、越需要跨层判断的任务，越应交给高阶模型或作者；越可复核、越机械、越格式化、越局部的任务，越可以交给中低阶模型。**

也就是说，不同任务的风险不一样。

高风险任务包括：

- 判断项目主攻方向；
- 修改核心定义；
- 决定某个概念是否进入主线；
- 对外定位和宣传语；
- 理论边界与降强；
- 商业化路线；
- 版本治理规则；
- AI 协作协议本身。

这类任务一旦做错，会影响后续大量文本和工具，因此必须由高阶模型或作者最终裁定。

低风险任务包括：

- 提取标题；
- 整理表格；
- 生成摘要草稿；
- 检查错别字；
- 生成目录；
- 批量改格式；
- 按模板填写字段；
- 对长文做初步切片。

这类任务即使出错，也容易被上级检查发现，可以交给中低阶模型完成。

因此，最重要的分工原则是：

```text
高阶模型少做量，多做判断；
中阶模型多做执行，但不定方向；
低阶模型只做可检查的机械任务；
所有模型输出都必须可复核。
```

## 13.3 智能体等级划分

为了便于调度，可以把参与项目的智能体分为四级。

### L4：战略与总架构智能体

这一层由最强模型或作者本人承担。

职责是：

- 判断项目主攻方向；
- 制定长期路线；
- 定义核心术语；
- 识别主矛盾；
- 处理跨卷结构；
- 决定哪些内容进入主线；
- 设计多智能体协作协议；
- 做最终验收。

它适合处理的问题是：

```text
这件事该不该做？
这件事放在哪一卷？
这句话是不是过强？
这个专题是不是该升主线？
这个方向是不是当前主攻？
这个工具包是否符合战略路线？
```

L4 的输出不一定最长，但必须最稳。它的价值在判断力，而不是字数。

### L3：高级研究与审稿智能体

这一层可以由强模型承担，也可以由经过严格提示词约束的中高阶模型承担。

职责是：

- 阅读多个文件后做结构审查；
- 给专题草稿做降强和补桥；
- 对中阶模型输出做二次复核；
- 生成章节草案；
- 设计案例分类法；
- 提出反方意见；
- 检查理论是否滑向万能化。

它适合处理的问题是：

```text
这份草稿哪里混层？
这段论证缺哪座桥？
这组任务怎么拆给低一级模型？
这篇文章是否符合主线？
哪些地方需要登记残差？
```

L3 可以写长文，但必须服从 L4 的主线。

### L2：中阶执行与初审智能体

这一层适合交给 GLM-4.7、普通商业大模型或本地较强模型。

职责是：

- 按模板整理材料；
- 从长文中提取卡片；
- 生成初稿；
- 做初步分类；
- 给案例打标签；
- 按指定标准做初评；
- 批量提取术语；
- 把评论整理成候选反馈；
- 生成低风险表格。

它适合处理的问题是：

```text
请按模板从这篇文章提取 10 张卡片；
请把这些评论分成 5 类；
请给每个案例标注使用场景；
请找出文中所有疑似过强句子；
请把这篇文章压成初版摘要；
请按评分表给这些任务做初评。
```

L2 可以“做很多”，但不能“最后决定”。它的输出必须标为候选。

### L1：机械处理智能体

这一层可以由更低成本模型、脚本或规则程序承担。

职责是：

- 去重；
- 改格式；
- 转 Markdown；
- 生成目录；
- 检查链接；
- 提取标题；
- 统计字数；
- 批量替换；
- 按固定字段输出 JSON；
- 做简单关键词匹配。

它适合处理的问题是：

```text
把这些标题转成表格；
把这批文本按模板分段；
检查哪些文件包含某个关键词；
把所有卡片字段补齐为空字符串；
把评论按时间排序。
```

L1 不应负责解释、判断和理论推进。它只负责机械性、可验证的处理。

## 13.4 任务等级划分

任务本身也要分级。可以分为五类。

### T5：战略级任务

定义：

> 影响项目方向、主线、定位、商业化和长期系统结构的任务。

例子：

- 项目价值路线图；
- 项目对外定位；
- 主攻方向选择；
- 五卷关系重排；
- 商业化顺序；
- 多智能体体系设计；
- 核心方法产品化路线。

执行要求：

```text
必须由 L4 主导；
L3 可以提供审稿和反方；
L2 只能收集材料；
L1 只能做格式和检索。
```

### T4：主线级任务

定义：

> 影响核心术语、章节主线、专题边界和理论强度的任务。

例子：

- 新增核心专题；
- 修改术语表；
- 判断某个概念是否过强；
- 给主书稿补关键桥；
- 处理外部批评；
- 决定是否登记残差。

执行要求：

```text
L4 或 L3 主导；
L2 可生成候选稿；
必须由 L4/L3 验收后才能回写主线。
```

### T3：产品化任务

定义：

> 把理论转成卡片、模板、提示词、案例、工具入口的任务。

例子：

- 10 张方法卡；
- AI 输出审查表；
- 行动复盘表；
- 真实案例库；
- 用户反馈模板；
- 网站工具包入口。

执行要求：

```text
L4 定标准；
L3 设计结构；
L2 批量生产初稿；
L1 做格式清洗；
L3/L4 抽样验收。
```

### T2：整理级任务

定义：

> 对已有材料做分类、摘要、索引、标签、表格和映射的任务。

例子：

- 提取章节摘要；
- 从评论中归类反馈；
- 建立术语索引；
- 给案例打标签；
- 做文件映射表；
- 生成阅读路径草案。

执行要求：

```text
L2 主导；
L1 辅助；
L3 抽样检查；
错误率高则退回重做。
```

### T1：机械级任务

定义：

> 低判断、强模板、可脚本化的任务。

例子：

- 格式转换；
- 链接检查；
- 标题提取；
- 字数统计；
- 文件名规范化；
- 目录生成；
- JSON 字段补齐。

执行要求：

```text
L1 或脚本完成；
L2 检查；
无需占用高阶模型，除非出现异常。
```

## 13.5 任务分配矩阵

| 任务类型 | L4 战略架构 | L3 高级审稿 | L2 中阶执行 | L1 机械处理 |
|---|---|---|---|---|
| T5 战略级 | 主导 | 反方/审稿 | 收集材料 | 格式辅助 |
| T4 主线级 | 裁定 | 主导/审稿 | 候选稿 | 格式辅助 |
| T3 产品化 | 定标准 | 设计结构/验收 | 批量初稿 | 清洗字段 |
| T2 整理级 | 抽检 | 抽检/纠偏 | 主导 | 辅助 |
| T1 机械级 | 不参与 | 不参与 | 检查 | 主导 |

这个矩阵的核心含义是：

> **低阶模型可以参与高阶任务，但只能做材料准备；高阶模型可以检查低阶任务，但不应长期占用在低价值机械工作上。**

## 13.6 GLM-4.7 适合做什么

如果把 GLM-4.7 作为 L2 中阶执行模型，可以给它安排以下任务。

### 适合任务

1. **卡片初稿提取**

从指定文章中提取方法卡，按固定字段输出。

2. **评论初步归类**

把评论分成进入反馈、结构修订、现实接口、误解风险、候写建议等类别。

3. **案例初步整理**

把用户提供的案例按“原问题、使用方法、结果、残差”格式整理。

4. **过强句子扫描**

找出疑似“解释一切”“必然”“完全证明”“本质就是”等高风险句子。

5. **摘要和导读草案**

为文章生成初版摘要、关键词和阅读建议。

6. **任务初评**

按评分表给任务打初步评级：价值、难度、风险、可复核性、是否适合低阶模型。

7. **模板填充**

把已有内容按指定表格或 JSON 字段填入。

### 不适合任务

GLM-4.7 不应负责：

- 最终决定项目定位；
- 修改核心定义；
- 宣布某个理论已经成立；
- 决定商业化方向；
- 独立重写主线章节；
- 直接回写术语表；
- 判断某个跨学科类比是否可升为机制；
- 对外发布最终版本。

如果必须让它处理这些任务，也只能让它输出：

```text
候选意见；
风险点；
可选方案；
需要上级复核的问题。
```

## 13.7 给 L2 模型的标准任务单

交给 L2 模型的任务，必须写成标准任务单，不能只说“帮我整理一下”。

标准格式如下：

```text
【任务类型】
T1 / T2 / T3 / T4 / T5

【你的角色】
你是 L2 中阶执行智能体，只负责生成候选结果，不做最终裁定。

【输入材料】
粘贴文本 / 文件路径 / 章节名称 / 评论列表

【目标】
本次要产出什么。

【禁止事项】
不能改核心定义；
不能自行升格结论；
不能删除残差；
不能把候选稿写成最终稿。

【输出格式】
严格按表格 / JSON / 分节 Markdown 输出。

【评分标准】
完整性、准确性、边界感、可复核性、可回写性。

【最后必须附上】
1. 不确定点；
2. 疑似风险；
3. 需要上级复核的问题。
```

这个格式的目的，是防止中阶模型越权。

## 13.8 给 GLM-4.7 的通用系统提示词

可以给 GLM-4.7 使用下面这段通用提示词：

```text
你是“差结构学习法”项目中的 L2 中阶执行智能体。

你的职责不是最终判断理论方向，而是按上级给出的标准完成可复核的工程任务：整理、提取、分类、初稿、初评、表格化、模板化。

你必须遵守：
1. 不修改核心定义；
2. 不把候选判断写成最终结论；
3. 不把跨领域类比写成已证明机制；
4. 不删除残差、例外和不确定点；
5. 不擅自改变卷册定位；
6. 不为了完整而扩写无关内容。

你输出时必须包含：
- 主结果；
- 不确定点；
- 风险点；
- 需要 L3/L4 复核的问题。

如果任务需要战略判断、核心术语裁定、主线重排、商业化决策或最终发布判断，你必须明确说：这超出 L2 权限，只能提供候选意见。
```

## 13.9 L2 任务初评表

每个任务在交给中低阶模型前，应先做初步评级。

| 维度 | 1 分 | 3 分 | 5 分 |
|---|---|---|---|
| 价值 | 可有可无 | 有局部价值 | 直接服务主攻方向 |
| 难度 | 机械整理 | 需要理解上下文 | 需要跨层判断 |
| 风险 | 错了易发现 | 错了会造成局部误导 | 错了会影响主线 |
| 可复核性 | 很容易复核 | 抽样可复核 | 很难复核 |
| 适合 L2 程度 | 不适合 | 可做候选稿 | 非常适合 |

注意：这里“适合 L2 程度”的高分，不等于任务简单，而是表示它适合交给 L2 做候选产出。

最终可以给出一个任务评级：

```text
L2-A：非常适合中阶模型执行，低风险、高复核性；
L2-B：可以执行，但必须抽样复核；
L2-C：只能做材料准备，不能做结论；
L2-X：不应交给 L2，必须由 L3/L4 处理。
```

## 13.10 不同任务的工程目标

为了让低一级模型有清楚目标，任务要尽量工程化。

### 卡片提取任务

工程目标：

```text
从一篇文章中提取 8-12 张方法卡。
每张卡包含：标题、主句、误区、动作、AI 提示词、适用场景、回原文位置。
```

验收标准：

- 不丢主线；
- 不新增无来源观点；
- 每张卡能单独使用；
- 每张卡有明确动作。

### AI 输出审查任务

工程目标：

```text
把一段 AI 输出拆成事实、推断、类比、建议、风险和待验证点。
```

验收标准：

- 事实和推断不混；
- 类比不写成证明；
- 风险点明确；
- 有下一步验证动作。

### 案例整理任务

工程目标：

```text
把一个现实案例整理成：原问题、差异轴、边界、行动、反馈、结构增量、残差。
```

验收标准：

- 案例不被拔高；
- 结果不夸大；
- 残差保留；
- 能用于案例库。

### 评论整理任务

工程目标：

```text
把评论分成：进入反馈、结构修订、现实接口、误解风险、候写建议、噪声。
```

验收标准：

- 不把批评清洗成赞美；
- 不把噪声全删；
- 高价值反馈有回写建议；
- 不确定项单独列出。

### 术语扫描任务

工程目标：

```text
扫描某篇文章中疑似未定义、过强或混层术语。
```

验收标准：

- 标出原句；
- 说明风险类型；
- 给出降强建议；
- 不直接替作者最终裁定。

## 13.11 上级复核流程

所有 L2/L1 输出都必须经过复核。复核分三级。

### 快速复核

适用于低风险任务。

检查：

- 字段是否完整；
- 是否按格式输出；
- 是否明显偏离任务；
- 是否遗漏禁止事项。

### 抽样复核

适用于批量任务。

做法：

- 随机抽 20%；
- 检查准确率；
- 若错误率超过 15%，整批退回；
- 若错误集中在同一类型，补充规则后重跑。

### 深度复核

适用于主线、专题、卡片和案例库。

检查：

- 是否忠于主线；
- 是否混淆层级；
- 是否把候选写成结论；
- 是否删除残差；
- 是否能被回写或产品化。

复核结果分为：

```text
通过：可进入下一步；
小修：局部修改后可用；
退回：需要重做；
升级：交给 L3/L4 处理；
废弃：不进入项目。
```

## 13.12 交接格式

每个智能体完成任务后，必须按统一格式交接。

```text
【任务编号】

【任务等级】
T1/T2/T3/T4/T5

【执行模型】
如 GLM-4.7

【输出摘要】
三到五句话说明完成了什么。

【主结果】
正文、表格、JSON 或卡片。

【不确定点】
列出模型自己拿不准的地方。

【风险点】
列出可能错误、过强、遗漏或混层的位置。

【建议复核级别】
快速复核 / 抽样复核 / 深度复核

【是否可直接回写】
是 / 否 / 只能作为候选
```

没有这份交接格式的输出，不应直接进入项目。

## 13.13 多级系统的工作流

完整工作流如下：

```text
L4 定主攻方向
→ L3 拆成任务包
→ L2 执行候选产出
→ L1 做格式和批处理
→ L2 自检
→ L3 复核
→ L4 裁定关键内容
→ 回写站点 / 书稿 / 工具包
```

## 13.16 这一章的压缩结论

多级智能体分工体系的本质，是把智能、成本、风险和复核性重新匹配。

如果压成一句话，就是：

> **强模型负责方向、边界和验收；中阶模型负责批量候选产出；低阶模型负责机械处理；所有输出必须带不确定点、风险点和复核等级。**

这样，本项目才能从“作者 + 一个强 AI”推进到“作者牵引、多模型分工、持续生成、可控回写”的协作系统。
