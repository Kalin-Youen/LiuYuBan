# 附录 第二轮任务包：术语收束、案例接口与轻量入口

> 适用对象：L2 / L3 / L4
> 状态：第二轮派工单
> 主攻目标：把第一轮候选收束成可正式使用的核心接口，不再横向扩写

## 一、总原则

```text
先收束核心，再扩展外围；
先做真实接口，再做漂亮表达；
先保留候选，再谈正式收录；
先让人能用，再让体系更完整。
```

本轮只做四件事：

1. 收束术语表；
2. 扩展过强句子扫描；
3. 建立真实案例接口；
4. 定义评论回写接口与轻量入口。

## 二、P0-01 术语候选压缩为核心术语表

**任务等级**：T3  
**适合模型**：L3 主导，L2 出候选  
**输入材料**：

- [附录_L2产出_C2_术语候选表.md](./附录_L2产出_C2_术语候选表.md)
- [专题_差异如何变成可用落差_从物理势差到认知问题再到文明结构.md](./专题_差异如何变成可用落差_从物理势差到认知问题再到文明结构.md)
- [第13章 多级智能体分工体系](./第13章_多级智能体分工体系_让强模型做判断让中低阶模型做工程.md)

**预期输出**：

- 核心术语表 15-20 个；
- 术语依赖关系；
- 哪些术语保留为桥接解释；
- 哪些术语只保留为工作假说；
- 哪些候选暂不收录。

**验收标准**：

- 每个核心术语都有明确含义；
- 术语数量被压住，不再一次性全收 37 个；
- “流/流动”“锚点包”“版本快照”等争议项有明确去向；
- 结果能直接回写术语表或附录说明。

**复核方式**：L3 深度复核，L4 裁定正式收录数

**给执行模型的提示词**：

```text
你是差结构学习法项目中的 L2 中阶执行智能体。

任务：请把《术语候选表》压缩成核心术语表第一版。

要求：
1. 只保留 15-20 个最核心、最稳定、最能支撑主线的术语；
2. 给每个术语标注：较稳定义 / 桥接解释 / 工作假说 / 待定义；
3. 标出术语之间的依赖关系；
4. 标明哪些术语应暂时保留为候选，不进入正式术语表；
5. 对“流/流动”“锚点包”“版本快照”等争议项单独说明处理方式；
6. 最后列出不确定点、风险点、需要 L3/L4 复核的问题。

输出格式：Markdown 表格 + 简短说明。
```

## 三、P0-02 过强句子扫描扩大到主书稿与白话卷

**任务等级**：T2/T3  
**适合模型**：L2 扫描，L3 复核  
**输入材料**：

- [研究文稿/07_书稿](./)
- [研究文稿/07_书稿/白话卷](./白话卷/)
- [研究文稿/07_书稿/专题_*.md](./)

**预期输出**：

- 主书稿和白话卷中的过强句子候选；
- 每条候选的原文、来源、风险类型、降强方向；
- 哪些句子适合保留为极限推演，哪些必须降强。

**验收标准**：

- 不能只扫专题；
- 不能只列信号词；
- 需要保留上下文；
- 只输出候选，不做最终删改。

**复核方式**：L3 逐条复核高风险项

**给执行模型的提示词**：

```text
你是差结构学习法项目中的 L2 中阶执行智能体。

任务：请把扫描范围扩大到主书稿和白话卷，继续找过强句子。

扫描信号包括：
必然、证明了、本质就是、绝对、完全、所有、同一机制、无条件成立、不需要验证、解释一切、不会统一、完全排除等。

要求：
1. 每条候选必须给出原句、文件名、所在上下文、风险类型、为什么有风险、建议降强方向；
2. 对于物理层极限判断，允许保留为候选，但要说明语境；
3. 对于“解释一切”一类说法，必须单独标记高风险；
4. 只输出候选，不要直接改正文；
5. 最后列出不确定点、风险点、需要 L3/L4 复核的问题。

输出格式：Markdown 表格。
```

## 四、P0-03 建立真实案例接口

**任务等级**：T3  
**适合模型**：L3 主导，L2 填充模板  
**输入材料**：

- [附录_L2产出_P101_案例库结构模板.md](./附录_L2产出_P101_案例库结构模板.md)
- [附录_L2产出_A1_10张核心方法卡.md](./附录_L2产出_A1_10张核心方法卡.md)
- [附录_L2产出_A2_AI输出审查表.md](./附录_L2产出_A2_AI输出审查表.md)

**预期输出**：

- 真实案例征集表；
- 轻量版案例模板；
- 失败案例特别填写项；
- 真实案例与模拟案例的分离规则。

**验收标准**：

- 不把模拟案例冒充真实案例；
- 真实案例字段少而清楚；
- 失败案例能单独被看见；
- 普通用户能直接提交。

**复核方式**：L3 设计，L4 指定正式接入口

**给执行模型的提示词**：

```text
你是差结构学习法项目中的 L2 中阶执行智能体。

任务：请把案例库模板改造成“真实案例接口”，不要编造真实案例。

要求：
1. 设计一个轻量版案例征集表，字段尽量少，但必须能收集到关键结构；
2. 设计一个完整版案例模板，用于后续深描；
3. 明确区分真实案例、模拟案例和演示案例；
4. 失败案例必须单独标注，并优先保留；
5. 给出用户可以直接复制提交的填写格式；
6. 最后列出不确定点、风险点、需要 L3/L4 复核的问题。

输出格式：Markdown。
```

```text

最后做 P0-05 轻量入口版。
```
