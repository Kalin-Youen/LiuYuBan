## 卡片 1：四词拆开法

| 字段 | 内容 |
|---|---|
| **卡片标题** | 四词拆开法 |
| **一句话主句** | 差异、差结构、落差、可用落差是四个层级，不能混写。 |
| **反例/误区** | 把"差异"直接写成"落差"，把"不同"直接当成"可用"，跳过边界、耦合和路径直接说"这个差距可以利用"。 |
| **使用动作** | 遇到任何"不同"时，依次追问四步：这只是差异吗？它有排布方式吗？排布出现方向性和强弱了吗？系统能调用它吗？ |
| **AI 提示词** | `请帮我检查以下文本中，"差异""差结构""落差""可用落差"四个词是否被混用。对每个混用位置，说明应该用哪个层级。` |
| **适用场景** | 理论写作审查、论文修改、概念辨析、项目讨论中的概念纠偏 |
| **风险提示** | 拆分过细可能让简单问题复杂化；四个层级是判断顺序，不是所有场景都必须走完全程。 |

> 原文依据：第二章"四个词必须拆开"，核心句："差异不是孤立对象的神秘属性，而是比较关系"

## 卡片 2：可用落差判据式

| 字段 | 内容 |
|---|---|
| **卡片标题** | 可用落差判据式 |
| **一句话主句** | 可用落差 ≈ 势差 × 耦合 × 路径 × 效率，任一因子为零则不可用。 |
| **反例/误区** | 有差异就认为可用；看到"机会"就认为能抓住；看到"信息差"就认为能变现。 |
| **使用动作** | 对任何"机会""差距""优势"，逐一检查四个因子：势差有多大？系统能接上吗？有释放通道吗？转换效率够吗？任一为零即不可用。 |
| **AI 提示词** | `请用"势差×耦合×路径×效率"框架分析以下差异是否可用。对每个因子给出0-5分评估，并指出哪个因子是瓶颈。` |
| **适用场景** | 决策评估、机会判断、项目选择、资源分析 |
| **风险提示** | 此式是判据式，不是普适物理公式；四因子之间可能有非线性交互，简单相乘只是工作记号。 |

> 原文依据：第二章第4节，B_avail ~ ΔΦ·C·P·η

## 卡片 3：总量≠可用量

| 字段 | 内容 |
|---|---|
| **卡片标题** | 总量≠可用量 |
| **一句话主句** | 能量总量可以守恒，但可用能会在不可逆过程中下降；资源多不等于可用资源多。 |
| **反例/误区** | 以为总能量大就等于可做功多；以为资源丰富就等于可调用资源多；以为信息量大就等于可用信息多。 |
| **使用动作** | 评估任何资源时，区分两个问题：总量是多少？其中有多少还能被调用？用"自由能/火用"思维替代"总量"思维。 |
| **AI 提示词** | `请帮我区分以下论述中的"总量"和"可用量"。对每项资源，分别评估总量和可被调用的比例。` |
| **适用场景** | 资源评估、能量分析、组织管理、信息筛选 |
| **风险提示** | 物理层的自由能和火用是严格概念，迁移到社会/认知领域时只是类比框架，不是同机制证明。 |

> 原文依据：第三章，dE/dt=0 但 dB/dt≤0

## 卡片 4：落差来源五分法

| 字段 | 内容 |
|---|---|
| **卡片标题** | 落差来源五分法 |
| **一句话主句** | 可用落差来自五个来源：初始条件、边界条件、约束结构、外部输入、涨落放大。 |
| **反例/误区** | 只看一种来源（如只看"外部机会"），忽略约束保存的落差或初始条件遗留的落差；把涨落放大当成常态。 |
| **使用动作** | 诊断"为什么没有落差"或"落差从哪来"时，逐一检查五个来源：系统一开始就不均匀吗？边界在持续输入吗？有约束在保存落差吗？有外部补给吗？有微小差异被放大了吗？ |
| **AI 提示词** | `请从五个来源（初始条件、边界条件、约束结构、外部输入、涨落放大）分析当前系统为什么缺乏可用落差，或可用落差从何而来。` |
| **适用场景** | 组织诊断、学习瓶颈分析、系统设计、竞争分析 |
| **风险提示** | 五来源是分类框架，不是因果方程；涨落放大只在特定不稳定条件下发生，不能泛化为"微小差异总会放大"。 |

> 原文依据：第五章

## 卡片 5：问题化判断法

| 字段 | 内容 |
|---|---|
| **卡片标题** | 问题化判断法 |
| **一句话主句** | 问题 = 差异进入主体目标、模型和行动接口后的可用认知落差。 |
| **反例/误区** | 把所有"不知道"都当成问题；把别人的问题当成自己的问题；把信息差直接当成认知落差。 |
| **使用动作** | 判断一个差异是否值得处理，检查三要素：有目标状态吗？主体能识别这份差异吗？有行动路径吗？三要素缺一则不是有效问题。 |
| **AI 提示词** | `请判断以下差异是否构成"问题"。按三要素分析：1)目标状态是什么？2)主体能否识别？3)有无行动路径？缺哪项就标明。` |
| **适用场景** | 学习规划、研究方向选择、AI 提问设计、问题优先级排序 |
| **风险提示** | "问题化"是桥接解释，不是已证明的认知科学机制；Q=Recognize(G-R|H,E,A) 是工作记号，不是标准认知模型。 |

> 原文依据：第八章

## 卡片 6：学习链诊断法

| 字段 | 内容 |
|---|---|
| **卡片标题** | 学习链诊断法 |
| **一句话主句** | 学习真正发生时走的是：差异→问题→行动→反馈→模型更新，断链即失败。 |
| **反例/误区** | 以为信息差自动转化为认知行动；以为"学了"等于"会了"；以为看了答案等于解决了问题。 |
| **使用动作** | 诊断学习失败时，逐节检查链条：差异被识别了吗？→形成问题了吗？→有行动吗？→有反馈吗？→模型更新了吗？断在哪节，就修哪节。 |
| **AI 提示词** | `请用"差异→问题→行动→反馈→模型更新"链条诊断我的学习瓶颈。对每个环节判断是否断裂，并给出修复建议。` |
| **适用场景** | 学习诊断、注意力管理、知识管理、培训设计 |
| **风险提示** | 五环节是诊断框架，不是认知科学的完整模型；实际学习中可能有并行和回路，不总是线性链条。 |

> 原文依据：第九章，D→Q→A→F→M'

## 卡片 7：耗散三分类

| 字段 | 内容 |
|---|---|
| **卡片标题** | 耗散三分类 |
| **一句话主句** | 耗散分正向（生成结构）、维持性（保持运转）、负向（消耗无结构），耗散≠进步。 |
| **反例/误区** | 把忙碌当成成长；把消耗当成进步；以为"做了很多"等于"有了结构"；把维持性耗散当成浪费。 |
| **使用动作** | 对每项行动问：它留下了可复用结构吗？（正向）它维持了必要运转吗？（维持性）它消耗了资源但没留下任何可复用结果吗？（负向） |
| **AI 提示词** | `请把以下行动清单按正向耗散（生成结构）、维持性耗散（保持运转）、负向耗散（消耗无结构）分类，并计算大致比例。` |
| **适用场景** | 时间管理、项目复盘、组织效率评估、个人精力分配 |
| **风险提示** | 三类边界有时模糊；维持性耗散不一定是坏事，许多结构必须靠刷新维持；判断"正向"需要明确结构标准。 |

> 原文依据：第十章

## 卡片 8：结构沉淀五条件

| 字段 | 内容 |
|---|---|
| **卡片标题** | 结构沉淀五条件 |
| **一句话主句** | 行动结果变成结构需要：可识别、可维持、可调用、可复制、可反馈。 |
| **反例/误区** | 以为"做过了"就有结构；以为写了文档就等于沉淀了知识；以为制定了制度就等于改变了行为。 |
| **使用动作** | 对任何行动结果，逐一检查：能被重新找到吗？能在扰动中维持吗？能被再次使用吗？能被复制传递吗？能反过来改变后续行动吗？缺哪项就补哪项。 |
| **AI 提示词** | `请用"可识别、可维持、可调用、可复制、可反馈"五条件检查表评估以下行动结果是否沉淀为结构。对每个条件给出通过/未通过判断。` |
| **适用场景** | 项目复盘、知识管理、制度建设、AI 协作产出管理 |
| **风险提示** | 五条件是检查表，不是通用数学定律；ΔE_structure ~ R·M·U·F·T - L 是工作记号，泄漏项 L 常被低估。 |

## 卡片 10：差异可用十问

| 字段 | 内容 |
|---|---|
| **卡片标题** | 差异可用十问 |
| **一句话主句** | 判断差异是否可用，依次回答十个问题，答不出则最多只是启发。 |
| **反例/误区** | 跳过检查直接下结论；只回答前几个问题就认为差异可用；把"感觉有用"当成"已经可用"。 |
| **使用动作** | 面对任何"机会""差距""优势"，用十问逐一检查：1被比较对象？2差异轴？3参照状态？4边界？5耦合机制？6释放路径？7损耗成本？8反馈在哪？9结果如何封存？10失败条件？答不出的问题就是瓶颈。 |
| **AI 提示词** | `请用"差异可用十问"检查表分析以下差异是否可用。对每个问题给出明确回答，答不出的标明为瓶颈。` |
| **适用场景** | 决策评估、研究设计、项目立项、理论审查、机会判断 |
| **风险提示** | 十问是检查框架，不能替代领域专业判断；不同领域对"耦合""路径""反馈"的具体含义不同，需要领域知识填充。 |

> 原文依据：第十五章

## 不确定点

1. 卡片数量与覆盖度：10 张卡是否足够？第六节（宇宙尺度）和第十四节（博弈层）未单独出卡。

3. AI 提示词的可用性未经实际用户测试。
4. 物理概念迁移的边界标注是否足够醒目。
5. 吸收系数的操作化未验证。

## 风险点

1. 混层风险：卡片 2 的"判据式"可能被误读为通用物理公式。
2. 过强迁移风险：卡片 3、4、5 涉及物理层向认知/社会层的概念迁移，强度分级未充分体现。
3. 简化失真风险：原文六种误判只分散吸收了部分。
4. 卡片独立性 vs 链条依赖：单独使用某张卡时可能丢失上下文约束。
5. "可用落差"一词的歧义：用户可能跳过风险提示直接使用。
