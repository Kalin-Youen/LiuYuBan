# 多级智能体协作系统 · 配置说明

> 适用对象：L2 / L3 / L4 跨 Trae IDE 会话协作
> 状态：v0.1 候选版

---

## 快速开始

### 1. 打开两个 Trae IDE 会话窗口
- 会话 A：配置为 L2（执行型）
- 会话 B：配置为 L3（复核型）+ L4（裁定型）

### 2. 初始化协作队列

两个会话都先执行：

```bash
cd e:\files\code\哲学研究
mkdir -p .collaboration/queue
mkdir -p .collaboration/context
mkdir -p .collaboration/output
```

---

## 协作流程定义

### L2 会话（执行型）职责
- 读取任务队列
- 执行 T1/T2/T3 任务
- 写入输出到 `.collaboration/output/`
- 标记任务为待复核

### L3 会话（复核型）职责
- 读取输出队列
- 深度复核 T3/T4 任务
- 给出复核意见
- 标记任务为待裁定

### L4 会话（裁定型）职责
- 读取复核意见
- 最终裁定任务结果
- 决定回写位置
- 标记任务为完成

---

## 队列文件格式

### 任务队列格式（`.collaboration/queue/task_*.json`）

```json
{
  "id": "TASK_20260504_001",
  "level": "L2",
  "task_type": "P0-01",
  "status": "pending",
  "input": {
    "files": ["文件路径1", "文件路径2"],
    "prompt": "任务提示词"
  },
  "priority": "high",
  "created_at": "2026-05-04T00:00:00Z"
}
```

### 输出队列格式（`.collaboration/output/output_*.json`）

```json
{
  "task_id": "TASK_20260504_001",
  "status": "pending_review",
  "output_files": ["文件路径1"],
  "summary": "任务完成摘要",
  "uncertainties": [],
  "risks": [],
  "needs_review": true
}
```

---

## 状态流转

```
pending → executing → pending_review → pending_decision → completed
         ↓                      ↓                      ↓
      L2执行              L3复核                 L4裁定
```
