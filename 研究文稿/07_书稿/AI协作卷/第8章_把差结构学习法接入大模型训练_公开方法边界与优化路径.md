# 第8章 把差结构学习法接入大模型训练：公开方法边界与优化路径

## 8.1 先把一句总纲说清

如果把“差结构学习法”压成适用于大模型训练的一句话，可以写成：

> 大模型训练，不是把海量数据硬灌进参数，而是持续识别并缩小多种落差，让能力在边界内站住、在反馈中变稳、在验证里留下可复用结构。

这里所谓“落差”，不只是一种修辞。放到大模型语境里，它至少同时对应六类差：

1. **预测落差**：模型分布与数据分布之间的误差；
2. **能力落差**：模型当前会做的，和目标任务要求之间的差；
3. **行为落差**：模型会答的，和我们希望它按什么方式答之间的差；
4. **环境落差**：纯文本内看似正确的输出，和真实工具、代码、检索、外部世界反馈之间的差；
5. **安全落差**：模型会生成的内容，与制度、权限、风险边界之间的差；
6. **长程落差**：单步可用，与多轮、多工具、多阶段任务里仍能保持稳定之间的差。

一旦把这几类差拆开，大模型训练就不再只是“规模越大越好”的问题，而会变成一条更清楚的结构链：

**差异识别 → 边界设定 → 语料组织 → 预训练压缩 → 后训练塑形 → 环境反馈 → 验证回写**

这正是“差结构学习法”最适合接入大模型训练的地方。

## 8.2 截至 2026-04-21，主流模型公开了什么，没公开什么

先把现实边界说清。不是所有知名模型都公开了完整训练方法，而且公开程度差异很大。

| 模型/体系 | 官方公开程度 | 当前能稳妥拿到什么 | 当前不能假装知道什么 |
| --- | --- | --- | --- |
| `GLM-5.1` | 部分公开 | 智谱官方文档公开了能力、模式、接口与适用场景 | 截至 `2026-04-21`，我未查到智谱官方发布的 `GLM-5.1` 完整训练技术报告或可复现实验细节 |
| `GLM-4.5` | 较多公开 | 技术报告公开了 `23T` token 预训练、多阶段训练、综合后训练、专家迭代与强化学习 | 不能直接把 `GLM-4.5` 的全部配方当成 `GLM-5.1` 已确认沿用 |
| `Gemini 2.5` | 高层公开 | Google 官方公开强调了更强的 base model、改进后的 post-training、以及 reasoning 方向的增强 | 目前公开材料没有给出完整数据配比、token 规模、后训练细配方与可复现实验流程 |
| `Gemma 3` | 较多公开 | 技术报告公开了蒸馏、长上下文、`12T/14T` token 训练量、后训练中的知识蒸馏、强化学习微调、人工反馈、代码执行反馈与数学真值奖励 | 不能把开放权重模型的做法机械套到闭源 Gemini 主系上 |
| `Qwen3-Coder-Next` | 较多公开 | 技术报告公开了合成可验证任务、可执行环境中的反馈、mid-training 与 RL 的结合 | 不能把代码代理训练直接当作通用模型的一般配方 |
| `LearnLM` | 对“教人”取向公开较多 | Google 公开了适合学习场景的原则与评估方向，如控制认知负荷、促进主动学习、识别误解、适配水平 | 它更像教学行为与评测原则，不等于完整基础模型训练路线 |

这张表很重要，因为它直接决定我们能怎样“接入现有大模型方法论”。

更稳妥的做法不是假装已经看透每家闭源模型的全部训练细节，而是分成两层：

- **第一层：拿公开得比较清楚的技术报告，提炼出可以稳定复用的方法结构；**
- **第二层：对公开不足的闭源模型，只使用官方确认过的高层信息，不擅自补出不存在的细节。**

如果把这条边界守住，方法论就会稳很多。

## 8.3 现有主流大模型训练方法，压到一起其实就五步

把不同机构公开的方法压缩后，当前主流大模型训练大致都在重复五个动作。

### 8.3.1 先做广域预训练，压缩大规模差异

这一步的核心不是“背知识”，而是让模型先在大规模语料里学会基本分布、模式、共现关系和语言组织能力。

如果用差结构学习法来翻译，就是：

> 先让模型在极大范围内看见差异，再通过损失函数把这些差异压成可泛化的内部结构。

这一层通常对应：

- 大规模多域语料；
- 去重、清洗、质量过滤；
- 语言、代码、数学、知识等不同分布的配比；
- 长短上下文混合；
- 基础架构稳定训练。

GLM-4.5、Gemma 3 这类公开材料，都把这一层放得很重。它说明一个事实：**没有广域差异输入，后面的高阶行为很难真正站住。**

### 8.3.2 再做中训或专项塑形，把泛能力拉成目标能力

仅靠泛预训练，模型通常会“会很多，但不够稳”。因此很多体系都会加上中间阶段：

- continued pretraining；
- domain adaptation；
- long-context adaptation；
- reasoning/coding/tool-use adaptation；
- 蒸馏；
- 合成数据增强。

如果翻译成差结构学习法，这一步的作用是：

> 把原本宽而散的能力差，重新压到某一类任务边界内，让模型开始长出更明确的能力结构。

Gemma 3 的蒸馏、Qwen3-Coder-Next 的 agentic mid-training，都属于这一层。

### 8.3.3 再做后训练，把“会做”改成“按要求做”

后训练是当前大模型方法公开得最多、也最容易被误解的一层。它常见的手段包括：

- `SFT`：监督微调；
- `DPO / preference optimization`：偏好对齐；
- `RL`：强化学习；
- `rejection sampling`：拒绝采样；
- teacher-student distillation；
- process supervision；
- verifiable reward。

如果用差结构学习法来翻译，后训练的任务并不是重新学世界，而是：

> 把“能说出来”和“应该怎样说、怎样做、怎样拒绝、怎样推理”之间的行为落差，往我们希望的方向压。

所以，后训练真正塑造的，不只是答案内容，还有：

- 响应风格；
- 推理耐心；
- 安全边界；
- 工具使用方式；
- 何时该拒绝；
- 何时该承认不确定。

### 8.3.4 再接入环境反馈，把文本闭环改成世界闭环

这一层是近两年最值得重视的新变化。很多强模型已经不再满足于“在纯文本 benchmark 上看起来很强”，而是开始把模型放进：

- 代码执行环境；
- 浏览器/工具调用环境；
- 检索系统；
- 多轮代理任务；
- 真实反馈闭环。

Qwen3-Coder-Next 把可执行环境中的反馈写得比较直接，Google 在 Gemini 2.5 方向上也明显在往更强的 reasoning/agent 行为推进。

如果用差结构学习法来翻译，这一步是在处理：

> 语言内正确，与环境里真的成立之间的落差。

这一步一旦缺失，模型就很容易形成一种“文本稳态假象”：在话里很通顺，在现实里一碰就散。

### 8.3.5 最后做评测、治理和版本回写

成熟模型训练不会止于“训完上线”。后面还要持续做：

- benchmark 评测；
- 红队与安全评估；
- 真实用户错误聚类；
- 版本回写；
- 残差登记；
- 数据和奖励系统更新。

如果翻译成差结构学习法，就是：

> 让训练中的剩余落差获得名字、位置和下一步处理方式，而不是让它们假装消失。

这一步很像本项目里一直强调的“残差登记”和“缺陷进入生长程序”。

## 8.4 差结构学习法给出的一版“大模型训练方法论”

在吸收现有公开方法之后，可以把更适合本项目的训练方法论压成七步。

### 8.4.1 第一步：先定能力边界，不先喊通用智能

训练前最先要问的不是“它最后会不会像人”，而是：

1. 这次到底训练什么能力？
2. 这项能力的边界在哪？
3. 哪些失败算可接受，哪些失败必须被压住？
4. 这项能力最终靠什么验证？

这一步若不先做，后面就会出现最常见的混乱：数据越堆越多，目标却越来越糊。

### 8.4.2 第二步：把训练对象拆成可管理的差结构

与其平均堆料，不如先按“差”来整理训练对象。最少要拆开：

- 领域差：通识、代码、数学、工具、长程任务；
- 难度差：基础模式、复杂约束、多跳推理、长链代理；
- 反馈差：可验证、半可验证、纯偏好、纯主观；
- 风险差：普通任务、高风险任务、拒答任务、边界任务；
- 迁移差：只在本域有效，还是能外推到别域。

这样一来，训练就不再是把所有样本放平，而是按差结构排布。

### 8.4.3 第三步：预训练负责压缩广域差异，不负责一次性解决全部行为问题

预训练的职责应当被写得更克制：

- 它负责让模型形成广覆盖的内部压缩结构；
- 它不负责一次性解决价值、风格、使用习惯和任务协议问题；
- 它提供的是“底层可塑性”，不是最终行为形态。

这能防止一个常见误区：把预训练规模误当成一切能力的直接来源。

### 8.4.4 第四步：后训练不要只做“更像人”，而要做“更能在边界内站住”

后训练的目标，不应只是把输出修得更顺，而应明确地压三种差：

1. **正确性差**：答案对不对；
2. **过程差**：推理过程是否可检查、可复盘、可修正；
3. **边界差**：该拒绝时能否拒绝，该求证时能否求证，该调用工具时能否调用。

这意味着，后训练最好同时包含：

- 高质量 `SFT` 轨迹；
- 偏好优化或强化学习；
- 可验证任务奖励；
- 失败样本与反例样本；
- 明确的拒答与求证样本。

### 8.4.5 第五步：把环境接进来，不要让模型只活在文本里

如果目标是强工具模型、代码模型或代理系统，那么训练一定要尽快接入环境反馈。至少要让模型经历：

- 代码执行；
- 工具返回错误；
- 检索不到结果；
- 计划失败后的重试；
- 多轮约束下的修正。

因为真正稳的能力，不是在一次回答里显得漂亮，而是在失败后还能重新组织。

### 8.4.6 第六步：把“残差登记”制度化

训练完成后，至少要持续记录六类残差：

1. 常见幻觉类型；
2. 长程任务中断点；
3. 工具调用错误模式；
4. 边界失守样本；
5. 看起来推理但其实乱编的样本；
6. 高价值但尚未站稳的能力萌芽。

很多团队会做 error buckets、safety categories、benchmark deltas，本项目可以把它们统一理解为“残差登记”。

### 8.4.7 第七步：让训练目标从“模型变大”转向“落差闭环变完整”

如果把这套方法再压成一句更短的话，就是：

> 大模型训练的关键，不是单纯让参数继续膨胀，而是让预测、行为、环境、安全与长程任务之间的落差闭环越来越完整。

## 8.5 一个更可执行的优化版：差结构训练五层栈

如果要把上面的思路变成更可执行的工程框架，可以写成一套五层栈：

### 8.5.1 第一层：差结构数据栈

不是平均采样，而是按以下维度建数据簇：

- 知识簇；
- 推理簇；
- 代码簇；
- 工具簇；
- 拒答簇；
- 修正簇；
- 长程代理簇。

每个数据簇最好附三项标签：

1. 目标能力；
2. 验证方法；
3. 主要失败模式。

### 8.5.2 第二层：阶段化目标栈

用不同阶段去压不同落差：

- 预训练优先压 `预测落差`；
- 中训优先压 `能力落差`；
- 后训练优先压 `行为落差` 与 `安全落差`；
- 环境训练优先压 `环境落差` 与 `长程落差`。

不要试图在同一个阶段平均解决一切。

### 8.5.3 第三层：奖励与验证栈

奖励系统不应只靠单一人工偏好，而应分层：

- 人工偏好：控制风格和可接受性；
- 真值奖励：数学、代码、检索、工具结果；
- 过程奖励：是否做了必要检查、是否自我修正；
- 风险惩罚：越界调用、伪装确定、危险建议。

也就是说，**奖励本身也应被看作一种差结构。**

### 8.5.4 第四层：残差回写栈

每次上线后，把错误重新归入：

- 数据需要补什么；
- 奖励需要改什么；
- 验证缺了什么；
- 工具接口哪里太脆；
- 哪些能力不该继续硬推。

这会让训练从“版本替换”变成“结构生长”。

### 8.5.5 第五层：治理边界栈

最终还要有一层专门处理：

- 权限；
- 法律；
- 伦理；
- 领域风险；
- 用户误用；
- 模型自信错觉。

因为模型强不强，从来不是单一能力问题，而是“能不能在正确边界里稳定工作”的问题。

### 8.5.6 为什么数据量不如模型的人，仍然主导信息流朝向

这里有一个很值得保留的判断：人类个体能接触和记住的数据量，当然远远比不过大模型；但现实中的信息流朝向，依然主要由人类社会在决定。这个现象本身提示我们，`数据量优势` 不等于 `方向控制权`。

更底层、也更硬的控制层，通常不在“谁记得更多”，而在：

- 谁决定采什么数据、不采什么数据；
- 谁设目标函数、奖励函数和验收标准；
- 谁提供电力、芯片、算力、机房和网络；
- 谁控制工具接口、权限边界和部署环境；
- 谁能把输出接进制度、市场、法律和组织流程。

如果把这件事翻译成差结构学习法的语言，就是：上层生成能力再强，也仍然受制于更底层的能量、材料、接口、边界和组织结构。很多时候，真正握着“命脉”的不是信息量最大的那一层，而是决定输入、约束、筛选、验证和调用条件的那一层。

这也给“AI 是否会脱离人类掌控”一个更稳妥的当前判断：至少在可见的工程路径上，AI 很难脱离人类提供的物质底座、数据入口、目标设置、权限系统与制度环境而完全自立。这不是说风险会自动消失，也不是说治理可以放松；更准确的说法是，AI 的上层能力越强，越说明底层接口、权限和治理结构必须被设计得更清楚。

### 8.5.7 公开看“模型怎样成长”，最好拆成四层而不是写成自传

如果要回答“像 ChatGPT 或 OpenAI 这类模型是怎样成长的”，更稳的做法不是编一段拟人化经历，而是按公开材料把它拆成四层：

1. `基础能力层`：预训练先在大规模数据里压缩语言、知识、代码与推理模式的广域差异。OpenAI 在 `GPT-4` 研究页里明确提到，很多能力主要来自 pre-training。
2. `行为塑形层`：后训练负责把“会说”拉成“按要求说、按边界做”。OpenAI 在 `gpt-oss` 的公开介绍里，把这一层写得很清楚：它包含 `SFT` 与高算力 `RL`，目标是让模型更贴近 `Model Spec`，并学会在回答前进行更合适的推理与工具使用。
3. `评测治理层`：模型不是训完就停。`Model Spec` 与配套 eval 的公开说明反复强调，真实输入、边界冲突、失败样式与新能力都会继续反向推动规则和评测更新。
4. `会话适配层`：系统消息、开发者约束、用户任务、工具接口、记忆设置与外部环境，会让模型在当前会话里呈现出不同工作状态。这属于运行时适配，不应直接等同于模型权重正在长期自我更新。

对闭源模型来说，精确数据配比、奖励配方、内部回写周期、哪些能力究竟出自哪一轮训练，通常都没有被完整公开。更诚实的写法，应把这些内容标成`基于公开材料的猜测`，而不是伪装成已经掌握的内部事实。

如果把这条经验翻译回“差结构学习法”，最值得借的不是某一家机构的神秘配方，而是这一整套阶段化治理方式：先让能力底盘形成，再让行为边界塑形，再让残差被评测抓住，最后再通过版本更新把旧失稳转成下一轮训练的起点。

## 8.6 对人类学习者来说，哪些能照搬，哪些不能照搬

这一章虽然主要写给大模型训练，但必须明确一句：**人不是参数模型，不能把机器训练配方原样照搬到人身上。**

人与模型至少有五个根本区别：

1. **人有身体、疲劳、情绪和动机，模型没有这一整套生命性约束。**
2. **人靠理解、意义和目标组织长期学习，模型主要靠损失与奖励更新。**
3. **人可用样本远少于模型，但人更擅长少样本抽象、类比与自我重组。**
4. **人需要睡眠、遗忘、间隔复习和社会反馈，模型训练流程不是这样工作的。**
5. **人学习的目标不只是输出正确，还包括人格、责任、判断和生活结构。**

所以，适用于人的不是“多喂 token”，而是：

- 先看差异；
- 再做比较；
- 再做小块压缩；
- 再做主动提取；
- 再靠反馈修正；
- 再靠迁移把结构写进自己。

这一部分会在白话卷的人类训练副本里单独展开。

## 8.7 这一章最值得保留的压缩结论

如果把这一章压成三句最适合保留的话，可以写成：

1. **主流大模型训练，公开得最完整的通常不是最强闭源模型，而是相邻的开放报告与技术论文。**
2. **差结构学习法最适合接入大模型训练的地方，在于它能把损失、奖励、环境反馈、安全边界与长程稳定性统一理解为多种“落差”的治理。**
3. **真正好的训练方法论，不是让模型只会在文本里看起来更强，而是让它在边界内、环境里、长任务中都更能站住。**
4. **数据量优势不自动带来方向控制权；训练目标、物质底座、接口权限和制度环境，通常仍握在更底层的治理层里。**
