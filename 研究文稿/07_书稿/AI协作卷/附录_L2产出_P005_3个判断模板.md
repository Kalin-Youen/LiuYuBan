## 模板 3：行动复盘结构表

```markdown
## 行动复盘结构表

**复盘对象**：（什么行动/项目/学习/决策）

### 一、行动输入

| # | 项目 | 内容 |
|---|---|---|
| 1 | 原始差异是什么？ |  |

| 3 | 可用落差在哪里？ |  |
| 4 | 采取了什么行动？ |  |

### 二、行动输出

| # | 项目 | 通过 / 部分通过 / 未通过 | 说明 |
|---|---|---|---|
| 1 | 结果可识别吗？ |  |  |
| 2 | 结果可维持吗？ |  |  |
| 3 | 结果可调用吗？ |  |  |
| 4 | 结果可复制/可传递吗？ |  |  |
| 5 | 结果可反馈吗？ |  |  |

### 三、耗散判断

| 类型 | 判断 | 依据 |
|---|---|---|
| 正向耗散（生成结构） | □ 是 □ 否 |  |
| 维持性耗散（保持运转） | □ 是 □ 否 |  |
| 负向耗散（消耗无结构） | □ 是 □ 否 |  |

### 四、残差与下一步

| # | 项目 | 内容 |
|---|---|---|
| 1 | 这次行动留下了什么可复用结构？ |  |
| 2 | 还有什么没解决？ |  |
| 3 | 下次行动如何增加结构沉淀？ |  |
| 4 | 需要什么新信息/反馈才能继续？ |  |

**综合判断**：
□ 这次行动沉淀了结构，可以继续
□ 这次行动维持了运转，但未新增结构
□ 这次行动主要是耗散，需要调整方向
```
