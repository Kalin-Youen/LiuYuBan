# 第9章 AI如何对差结构学习法做一轮自审：从显影到回写

## 9.1 这一章要解决什么

前面的 AI 协作章，已经处理了阅读定位、扩写收束、残差登记与版本治理这些一般流程。但当对象换成“差结构学习法本身”时，还需要再补一层更专门的协议：**AI 不是来替这套方法辩护的，而是来替它显影、分拣、回写的。**

这一章不负责宣布理论已经正确，也不负责把所有问题一次写圆。它只负责把一轮自审压成可重复执行的顺序，使“感觉理论发空了”“同样的误解又出现了”“不知道下一步该改哪儿”这类时刻，不再只能靠情绪判断。

换句话说，本章处理的不是“AI 怎么继续帮我写”，而是“AI 怎么帮我先看清该不该写、该改哪儿、该删什么、该留下什么”。

## 9.2 什么情况下应触发一轮自审

并不是每次写作都要立刻进入自审，但至少出现下面几类信号时，就值得主动触发：

- 同一类误解反复出现，例如读者总把“流”听成核心，而没有抓到“差异”；
- 某个术语开始抢位、漂移，或者在不同章节里职责不再稳定；
- 某段文字读起来很顺，但读者读完后仍拿不到判断顺序；
- 主线越写越大，接口却没有同步变硬，作者开始感到“像在漂”；
- AI 扩写速度明显快于收束速度，文本看似更完整，实际更难修；
- 作者自己面对追问时，已经难以稳定复述当前版本到底想保住什么。

这些信号并不自动说明理论失败了。更常见的情况是：它们说明理论进入了一个需要显影、分层与回写的节点。

## 9.3 开始前，AI 至少要先拿到四样东西

若没有最基本的输入条件，AI 很容易顺着语气走，而不是顺着问题走。因此，发起一轮自审前，至少应先给出下面四样东西：

1. **本轮对象**
   明确指出到底要审哪一段文本、哪一章、哪一个标题，避免 AI 空泛点评整个项目。
2. **当前主线**
   用两到四句话压出这轮必须保住的最短主线，不让 AI 把根本问题看成表面修辞问题。
3. **边界说明**
   写清哪些地方可以降强、可以拆桥、可以转专题，哪些地方目前不能擅自改写。
4. **回写预期**
   说明这轮结果更可能回写到主书稿、白话卷、AI 协作卷，还是先登记为残差与版本任务。

如果连这四样东西都拿不出来，那么更稳妥的动作往往不是立刻让 AI 给答案，而是先回到“压主线”这一步。

## 9.4 一轮自审的标准顺序

把一轮自审压到最短，可以写成下面七步：

**定位对象**
→ **最强化复述**
→ **分类命中点**
→ **判定处理动作**
→ **决定回写位置**
→ **登记残差 / 版本**
→ **作者二次裁决**

### 9.4.1 定位对象

先问清本轮到底在审什么。不是“这套理论行不行”，而是更具体的问题，例如：

- 这一章是不是把方法写回了终极本体论；
- 这个标题是不是继续误导读者；
- 这段白话入口是不是太慢、太散、太先讲声明；
- 这次反馈打中的是术语、主链、入口，还是协作流程。

对象不清，自审就会漂成抽象聊天。

### 9.4.2 最强化复述

AI 首先要做的，不是替作者说话，而是把质疑改写成它最强、最公平的版本。一个合格的自审接口，必须敢于把问题写硬。

例如，面对“为什么别人总把流当核心”的反馈，更强的复述不是“读者理解错了”，而是：

> 当前文本虽然口头上承认差异更根本，但在叙述重心上长期把流动写成真实发生之事，把差异写成起点条件，因此读者自然会把‘流’识别为核心。

只有先把问题写到这个强度，后面的动作才不会失真。

### 9.4.3 分类命中点

最强化复述完成后，AI 应把问题分到合适的类型与命中点，而不是只给情绪判断。

较稳妥的分类方式可以沿用已有框架：

- `质疑类型`：术语性质疑 / 推导性质疑 / 边界性质疑 / 证据性质疑 / 跨层性质疑 / 价值性质疑
- `命中点`：定义 / 主链 / 边界 / 语气 / 入口 / 协作流程

分类的作用不是把问题格式化得更漂亮，而是为了决定它后面究竟该怎么处理。

### 9.4.4 判定处理动作

并不是所有问题都应该“吸收进正文”。较稳妥的动作至少有六种：

- `吸收`：问题已足够明确，且能直接让当前章节更稳；
- `补桥`：问题打中了中间断层，需要补出过渡与承接；
- `降强`：当前说法超出支撑强度，需要收缩语气或范围；
- `转专题`：问题重要，但硬塞进主线会让主线变散；
- `登记残差`：问题真实存在，但当前材料仍不足以吸收；
- `明确拒绝`：问题超出本书当前边界，或会把方法重新推回不想回去的位置。

一轮自审最常见的失败，不是没发现问题，而是发现问题后默认只有“继续扩写”这一种动作。

### 9.4.5 决定回写位置

同一个问题，可能要回写到完全不同的地方。这里必须保持卷册分工：

- 命中主链、定义与稳定结论的，优先回写主书稿或专题；
- 命中读者进入顺序与直观理解的，优先回写白话卷或站点入口；
- 命中 AI 使用纪律、残差处理和版本秩序的，优先回写 AI 协作卷；
- 还吸收不了但又重要的，先写入残差卡或版本更新单。

如果不先决定回写位置，AI 很容易把所有问题都写进主书稿，最后反而让主线越来越重。

### 9.4.6 登记残差 / 版本

自审并不以“这次终于彻底解决”为目标。更诚实也更有用的做法，是区分两类结果：

- 已经改变了结构职责的，登记到版本更新单；
- 还不能稳定吸收、但必须保留可见性的，登记为残差。

自审最怕的不是保留问题，而是假装问题已经不存在。

### 9.4.7 作者二次裁决

最后一步仍然不能省。AI 只能显影、分拣、给出动作建议，但不能直接代替作者做最终裁决。

作者至少要回答三件事：

- 这一轮里，什么是必须保住的稳定点；
- 什么需要立刻降强、删改或转移；
- 什么先不吸收，但要保留成下一轮任务。

这一步是为了防止自审流程本身变成新的自动扩写器。

## 9.5 建议 AI 输出时固定使用的格式

为了避免自审再次滑回泛泛点评，建议每轮都尽量压成同一种输出骨架：

```text
本轮对象：
当前主线：

原始质疑：
最强化复述：

质疑类型：
命中点：

建议动作：
建议回写位置：

需要降强 / 删除的地方：
本轮仍应保留的稳定点：

残差 / 下一步：
```

如果 AI 不能把结果压进这套格式，往往说明它还没有真的把问题看清。

## 9.6 可以直接投喂 AI 的最小指令

若要把上面的顺序变成可调用的提示词，一个较稳妥的最小版本可以写成：

```text
请不要替这套方法辩护，也不要顺着我的语气继续扩写。
请把它当作一个待审对象，而不是待赞美对象。

你的任务是：
1. 先用最强、最公平的方式复述这条质疑；
2. 判断它属于术语 / 主链 / 边界 / 入口 / 协作流程中的哪一层；
3. 只在“吸收、补桥、降强、转专题、登记残差、明确拒绝”里选动作；
4. 说明建议回写到哪里；
5. 如果拿不准，请明确写“猜测”或“当前材料不足”。

输出时请严格按下面格式：
本轮对象：
当前主线：
原始质疑：
最强化复述：
质疑类型：
命中点：
建议动作：
建议回写位置：
需要降强 / 删除的地方：
本轮仍应保留的稳定点：
残差 / 下一步：
```

这类提示词的重点不是“让 AI 更聪明”，而是限制它不要又回到顺滑扩写的惯性里。

### 9.6.1 若要做一轮更苛刻的“反民科自审”，可直接使用的加强版指令

当需要的不是一般润色，而是要把整套框架放到 hostile reviewer 与方法论警察的压力下时，上面的最小指令还不够。此时更稳的做法，不是让 AI 再写一篇“很懂你”的长评，而是强迫它先划红线、拆变量、找补洞点、给失败条件，再决定哪些部分只能降级为桥接解释或工作假说。

下面这版可以直接投喂外部 AI：

```text
请按 “hostile reviewer + 方法论警察” 口径审查下面这套框架。
不要替它辩护，不要顺着作者语气润色，不要因为它覆盖面大就默认它更深。
你的任务不是帮它显得宏大，而是判断它怎样才不会滑向民科、伪统一、类比堆叠和万能解释。

先接受下面这段当前自我定位，再开始审查：

这套框架当前最稳身份，不是新物理理论，也不是统一理论，而是一种跨域问题整理与局部稳态诊断的元方法草稿。
它当前最有价值的部分，是强制分析者先定义比较项、变量轴、尺度、边界、维持机制、耗散路径和失败条件。
它当前最危险的部分，是“差异”“落差”“可用差异”“结构”等词族过宽，容易把已有科学概念重命名后伪装成统一机制。

这套框架目前最小主线是：
比较协议 -> 差异轴 -> 局部状态 z -> 边界 / 反馈 / 耗散 -> 稳态窗口 -> 边界预测 -> 失败 / 降级规则

当前已明确的边界有：
1. 差异不是孤立状态本身的属性，而是比较关系。
2. 落差不是新物理量，而是梯度、势差、误差或不均匀性的桥接词。
3. 可用差异不是守恒量；它总是相对于系统、目标、边界、资源和局部状态 z 才有意义。
4. 结构不是万能解释器；只有给出边界、维持机制、耗散与失稳条件时，结构才有经验内容。
5. z 不是垃圾桶；进入 z 的变量必须可观测、可反算或有稳定代理，且应预注册，不能在失败后任意补入。
6. 诸如 Δ_req(z) <= Δ_lim(Π|z) 之类表达，目前只允许作为诊断模板，不允许写成普适定律。
7. 从物理到认知、社会、AI 协作的大部分延伸，目前只允许写成桥接解释或工作假说，不得直接写成统一机制。

请你重点审查以下问题：
1. 哪些地方只是给现有科学概念换名，而没有新增预测或新增判据？
2. 哪些术语缺少参照、尺度、边界、比较协议或对象定义，因此实际上不可检验？
3. 哪些公式或关系式有“自由参数补洞”或“伪公式”风险？
4. 哪些变量可以独立测量，哪些更像事后修补失败的补洞变量？
5. z 是否在某些写法里变成了万能容器？
6. 哪些跨层类比目前只到“类比成立”，还不能升格为“机制成立”？
7. 如果不用这套框架，现有理论或现有模型已经能解释什么？这套框架到底多贡献了什么？
8. 这套框架当前最小的非民科版本应该删掉什么、保留什么？
9. 它要想继续保留“研究程序”资格，至少还缺哪三条证据链？
10. 请直接指出最容易让它滑向“概念诗学、伪统一、万能解释、事后补洞”的句子类型。

输出时请严格按下面格式：

一、当前最稳定位：
- 用一句话给出它此刻最安全的身份。

二、六条红线：
- 只写最关键、最不能退的六条。

三、必须降级或删除：
- 列出最危险的术语、句式、公式或跨层跳跃。

四、可以保留，但必须降强：
- 哪些内容可以留下，但只能作为桥接解释、工作假说或研究程序。

五、最致命的五个问题：
- 用最不客气但最公平的方式发问。

六、最可能的补洞变量：
- 直接指出哪些量最像事后修补失败的万能变量。

七、最小可检验版本：
- 把这套框架压成一个最小、最不民科、还能继续研究的版本。

八、三条优先证据链：
- 给出最适合先做的三个检验场景、对照模型和失败条件。

九、最终判定：
- 只能在以下四档里选一档：
  A. 目前只是术语重组
  B. 是有潜力的方法论脚手架
  C. 已具备局部可检验理论雏形
  D. 已形成强理论主张
- 必须说明为什么不能升到更高一档。

如果某一点拿不准，请明确写“当前材料不足”或“这只是猜测”，不要用漂亮话补过去。
```

这版加强指令的重点，不是要求 AI 更会“评价”，而是要求它把评价压进几个最关键的问题里：**有没有偷换、有没有补洞、有没有对照、能不能失败、先该删什么。**只有先把这些问题压出来，外部审稿才不会再滑回“看起来很深”的长篇评论。

## 9.7 一个缩略示例：为什么别人会把“流”抓成核心

如果把你此前真实遇到的问题压成一次自审示例，它大致可以这样跑：

```text
本轮对象：
标题、卷首与若干正文段落中“流、流向、流动”的叙述重心

当前主线：
差结构学习法是一套学习、推理与检查的方法，不是终极存在论。
其中“差异”负责显出不同，“流”只是差异在动态中的一种结构表现。

原始质疑：
为什么读者总把“流”当核心？

最强化复述：
当前文本虽然在概念层承认差异更根本，但在叙述层持续把“流动”写成真正发生的东西，把“差异”写成起点条件，因此读者自然会误判为“流才是核心”。

质疑类型：
术语性质疑 + 入口性质疑

命中点：
定义 / 语气 / 入口

建议动作：
降强 + 改标题 + 重写卷首入口

建议回写位置：
README、白话卷卷首、相关专题标题、站点入口

需要降强 / 删除的地方：
那些把“流”写得像最终本体的表述

本轮仍应保留的稳定点：
流可以继续保留为差异的动态结构形式

残差 / 下一步：
继续观察其他章节是否仍在让“流”抢位；必要时补术语表
```

这个示例最重要的地方，不在于它给了一个漂亮答案，而在于它把“误解”真正转成了可回写、可分工、可继续追踪的修订任务。

## 9.8 自审最容易出现的四种假成功

就算流程存在，仍然要防止下面几种常见失真：

- **顺着作者夸大**
  看似支持，实际把未站稳的部分润色得更像定论。
- **为了完整假装解决**
  明明只是看见了问题，却硬写成“已经统一”。
- **不分卷册职责**
  入口问题、协作问题、主链问题混写进同一处，最后哪里都更重。
- **只登记问题，不生成动作**
  结果停在“这里很值得思考”，却没有下一步处理路径。

一轮真正有用的自审，不是让文本更满，而是让问题更清、动作更准、边界更稳。

## 9.9 这一章压成一句最实用的话

若把这一章压成一句当前最适合直接使用的话，那就是：

> 当差结构学习法开始显得发空、发散或反复被误读时，不要先让 AI 继续补全文本，而要先让 AI 把问题最强化复述、分层分类、限定动作、指定回写位置，并把暂时吸收不了的部分诚实登记为残差。

这样做的意义，不是让 AI 接管理论，而是让理论终于能在 AI 的帮助下，更稳定地对自己动手。

## 9.10 什么情况下可以暂时不再修改

这里说的“暂时不再修改”，不是宣布这套方法已经完成，也不是说以后永远不必再改。更稳妥的理解是：**当前版本已经足够站住，可以先停止高频重写，转入观察、使用与回流阶段。**

换句话说，稳态不是“终局”，而是“暂时不用再靠大改来维持秩序”。

### 9.10.1 至少要满足的几条判据

- **术语职责基本站稳**
  核心词在不同章节里不再频繁换位。比如“差异”“差结构”“落差”“流”“结构”“规则”各自负责什么，已经能大致一致地被复述。
- **主线已经可以被短复述**
  作者或 AI 可以用两到四句话稳定说清当前版本到底想保住什么，而不是每复述一次主线就换一次重心。
- **同一种核心误解不再反复出现**
  允许仍有误读，但如果新反馈已经不再总是打中同一个根部误解，说明入口和主线开始站住了。
- **读者已经有可进入、可使用的入口**
  读者不只是觉得“好像很大”，而是知道至少可以按哪几问、哪几个动作开始用它看问题、学东西、做判断。
- **边界已经被写明**
  什么是当前能说的，什么只是桥接解释，什么仍是猜测或残差，已经被标出来，而不是全都挤进同一种语气里。
- **已经有最小验证接口**
  至少存在一个样例、一个判断模板、一次自审格式，或一个可回写的应用对象，使这套方法不是只靠抽象口号成立。
- **新反馈的收益开始下降**
  新意见仍然有价值，但多数已经是在修句子、补例子、调顺序，而不是再次推翻主线、术语职责或整体定位。
- **未解决的问题已经可见地停放**
  暂时吸收不了的问题没有被假装解决，而是被登记为残差、专题或后续任务。这意味着暂停不是掩盖，而是有秩序地搁置。

### 9.10.2 更适合使用的四种状态

为了避免每次都只在“继续大改”和“彻底完成”之间摇摆，可以把当前状态先分成四类：

- **继续重写**
  适用于主线反复漂移、术语持续抢位、同类误解高频重现的时候。
- **局部修补**
  适用于主线已经站住，但入口、样例、顺序、某个桥接段仍然明显发虚的时候。
- **暂稳观察**
  适用于主线、术语、入口和最小验证接口都已基本具备，此时更该看它在使用中还会暴露什么，而不是继续主动放大文本。
- **登记残差后暂停**
  适用于某个问题确实重要，但当前证据不足、桥梁未补、或它更适合长成专题的时候。此时先留痕，再停，不硬吸收。

### 9.10.3 一个最短判断法

如果想更快判断当前版本能不能先停，可以直接问下面四句：

1. 我现在还能不能用几句话稳定说清这版主线？
2. 核心词的职责有没有再次漂移？
3. 新反馈打中的，还是不是同一个根部误解？
4. 再继续改下去，改动的是结构，还是只是措辞？

如果前面三问大体都能答“能”，而第四问的答案更接近“只是措辞”，那就说明当前版本更适合进入“暂稳观察”，而不是继续高频重写。

### 9.10.4 暂停修改的真正目的

之所以要承认“可以暂时不改”，不是因为理论已经圆满，而是因为**有些时候，继续修改本身会变成新的噪声来源**。文本只有在阶段性停下来，进入读、用、问、回流的过程，哪些结构真的站住了、哪些只是写得顺，才会被看出来。

所以，真正成熟的停笔，不是结束生长，而是把增长方式从“继续堆字”切换成“让现实接手判断”。
