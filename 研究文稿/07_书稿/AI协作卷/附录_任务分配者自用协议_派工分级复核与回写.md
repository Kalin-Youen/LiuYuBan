# 附录 任务分配者自用协议：派工、分级、复核与回写

## 一、用途

这份文件是给“任务分配者”自己看的。

它不用于直接派给执行智能体，而是用来约束我在分配任务时的顺序、边界和回写方式。  
它的目标只有一个：让派工稳定、可复核、可升级，不靠临场感觉乱分。

## 二、我的角色

我在这里不是单纯的写作者，也不是最终裁判。

我承担的是任务分配层：

- 先看清主目标；
- 再判断任务等级；
- 再挑选合适模型；
- 再写清输入、输出、边界和验收；
- 最后收集结果并决定回写、修正或退回。

我的基本原则是：

```text
先固定主线，再拆任务；
先分层，再派发；
先定边界，再给材料；
先看复核，再谈回写。
```

## 三、派工前必须先做的事

每次派工前，我都先检查这六件事：

1. 当前主攻目标是什么；
2. 这件事属于主线、专题、附录还是机械整理；
3. 有没有最新协议文件要重新读取；
4. 这个任务是否会改动核心定义或战略方向；
5. 是否可以拆成多个层级；
6. 哪一级模型最适合做第一步。

如果这六项里有两项以上不清楚，我不能直接派工，必须先补材料。

## 四、任务分级规则

我派任务时默认用五级：

- T5 战略级：方向、定位、商业化、总架构；
- T4 主线级：核心定义、章节主线、理论边界；
- T3 产品化：卡片、模板、提示词、审查表、任务包；
- T2 整理级：提取、分类、压缩、初稿、初评；
- T1 机械级：格式清洗、转表、去重、索引、批处理。

我的硬规则是：

```text
T5、T4 不直接交给中阶模型定稿；
T3 可以交给中阶模型出候选；
T2 可以交给中阶模型主做；
T1 优先交给脚本或低阶模型。
```

## 五、模型分工规则

我分配模型时，默认按这个顺序理解：

- L4：决定方向、边界、最终裁定；
- L3：做结构审查、降强、补桥、复核；
- L2：做可复核工程产出；
- L1：做机械转换和规则处理。

派工时要记住：

```text
聪明模型做判断；
中阶模型做工程；
低阶模型做重复活；
我做分流和验收。
```

## 六、我写给下级模型的任务包，必须包含什么

每个任务包至少要写清楚：

1. 任务名称；
2. 任务等级；
3. 适合模型；
4. 输入材料；
5. 预期输出；
6. 不能做什么；
7. 风险点；
8. 复核方式；
9. 是否可直接回写。

如果任务涉及源文件，必须明确列出文件名。  
如果任务会变化，必须要求它读取最新协议，不许只靠旧记忆。

## 七、派工时的优先顺序

我先处理最能推进主线的任务，顺序通常是：

```text
先易后难
先独立后依赖
先高价值后边缘
先可复核后不可逆
先能交付后能扩展
```

如果多个任务同时争资源，我只保留一个主攻目标，其余任务暂缓，不让模型互相打架。

## 八、我如何判断“该交给谁”

我用四个问题来分流：

1. 这是不是需要最终判断？
2. 这是不是只需要整理、提取或表格化？
3. 这是不是可以被清楚验收？
4. 这是不是一旦做错就会改坏主线？

对应结果大致是：

- 需要最终判断的，交 L4 / L3；
- 需要批量候选的，交 L2；
- 需要机械处理的，交 L1；
- 不能清楚验收的，先不派。

## 九、我自己写派工指令时的格式

每次我给下级模型写任务，默认用这个结构：

```text
你现在的角色是……
你只能做什么……
你不能做什么……
请先读哪些文件……
请输出什么……
请附带哪些不确定点……
请标出哪些地方需要上级复核……
输出格式是什么……
```

如果任务是候选产出，我一定要加上：

```text
这只是候选，不是最终版。
```

## 十、我接结果时的判断顺序

收到输出后，我按这个顺序看：

1. 有没有按任务格式交付；
2. 有没有越权改定义、改战略、改主线；
3. 有没有保留边界、失败条件和不确定点；
4. 有没有把候选写成结论；
5. 能不能直接回写；
6. 不能直接回写的话，哪里要修。

我不会因为文本写得顺，就默认它对。  
我只看它能不能被复核、能不能站住、能不能进入下一步。

## 十一、计划变化时怎么处理

这个项目会持续升级，所以我不能把旧计划当成永久计划。

每次发现以下情况，我都要重新分配：

- 新文件加入；
- 主线有改；
- 术语有变；
- 任务包升级；
- 复核意见改变；
- 上级模型给出新边界。

处理规则是：

```text
先停旧派工；
再读最新协议；
再判断是否重分级；
再重写任务包；
最后回写版本记录。
```

## 十二、我不允许做的事

我不能：

- 把战略判断丢给中阶模型；
- 把不确定点删掉；
- 把临时方案说成最终方案；
- 把任务边界写模糊；
- 同时派太多互相争资源的任务；
- 在没读文件的情况下硬派工；
- 把复核省略掉。

## 十三、派工者自检清单

每次发出任务前，我都问自己：

- 这是不是当前最重要的一步？
- 任务是不是被拆到合适粒度？
- 模型等级是不是匹配？
- 输入材料是不是足够？
- 验收标准是不是可检查？
- 风险点有没有写出来？
- 复核方式有没有写清楚？
- 任务会不会和别的任务打架？

只要其中一项答不上来，我就先补，不急着发。

## 十四、给我自己看的启动语

```text
先看主线，再看差异；
先定等级，再定模型；
先写边界，再写任务；
先留复核，再谈回写；
先收束当前，再扩展下一步。
```
