## 提示词 1：帮我判断这个问题真正差异在哪里

```text
请帮我分析以下问题中真正的差异在哪里。

要求：
1. 先指出被比较的两个对象或状态是什么；
2. 再指出差异轴是什么（温度差、信息差、能力差、目标差、制度差等）；
3. 再判断这个差异是否形成了落差（有无方向性和强弱）；
4. 再判断这个落差是否可用（系统能否耦合、有无释放路径、效率如何）；
5. 如果差异不可用，说明缺哪个条件（边界？耦合？路径？效率？）；
6. 最后给出一个可以验证的下一步动作。

输出格式：
被比较对象：
差异轴：
是否形成落差：
落差是否可用：
缺失条件（如有）：
下一步验证动作：
```

## 提示词 2：帮我检查这段观点是否过强

```text
请帮我检查以下观点是否过强。

检查标准：
1. 有没有把桥接解释写成已证明结论；
2. 有没有把跨领域类比写成同一机制；
3. 有没有缺少边界条件的全称判断（"所有""必然""完全""本质就是"）；
4. 有没有把工作假说写成稳结论；
5. 有没有把局部有效写成普遍规律。

对每个发现的问题：
- 标出原句；
- 说明风险类型；
- 给出降强方向（不是直接改写，而是指出可以怎么收窄）；
- 标明是否需要更高级别复核。

如果这段观点没有过强问题，也请明确说"当前强度合理"并说明理由。

输出格式：Markdown 表格。
```

## 提示词 3：帮我把一个想法压成可验证行动

```text
请帮我把以下想法压成可验证行动。

要求：
1. 先用一句话说清这个想法的核心判断；
2. 再说明这个判断目前属于哪一层：事实 / 推断 / 类比 / 假说 / 建议；
3. 再说明如果这个判断是对的，最直接的验证动作是什么；
4. 再说明如果这个判断是错的，什么结果能让我看出来；
5. 再估算验证这个判断需要的最小成本（时间、注意力、资源）；
6. 最后给出一个不超过 30 分钟的最小验证动作。

输出格式：
核心判断：
判断层级：
验证动作（如果对）：
失败信号（如果错）：
最小成本：
30分钟验证动作：
```

## 提示词 4：帮我复盘这次行动有没有留下结构

```text
请帮我复盘以下行动是否留下了可复用结构。

检查清单：
1. 这次行动的结果能否被重新找到和识别？
2. 结果能否在一定扰动中维持？
3. 结果能否被再次调用？
4. 结果能否被复制或传递给他人？
5. 结果能否反过来改变后续行动？

对每个问题给出：通过 / 部分通过 / 未通过，并说明原因。

如果五项都未通过，说明这次行动可能只是一次性耗散。
如果部分通过，指出最容易补的一项，并给出补法。

最后回答：
- 这次行动是正向耗散、维持性耗散还是负向耗散？
- 下次行动如何增加结构沉淀的概率？

输出格式：Markdown 表格 + 综合判断。
```

## 提示词 5：帮我把 AI 输出按边界、证据、假设和残差重排

```text
请把以下 AI 输出按四类重新排列：边界、证据、假设、残差。

分类标准：
- 边界：这段输出在什么条件下成立，什么条件下不成立；
- 证据：哪些部分有可查证的来源或数据支撑；
- 假设：哪些部分是推断、类比或未经检验的桥接解释；
- 残差：哪些部分当前无法判断，需要保留但暂不能吸收。

要求：
1. 对 AI 输出的每一条关键判断，归入上述四类之一；
2. 如果一条判断同时包含证据和假设，拆开标注；
3. 对每条假设，给出一个可以检验它的动作；
4. 对每条残差，说明需要什么材料才能继续判断；
5. 最后给出综合判断：这段 AI 输出可以直接行动 / 需要补充信息 / 需要换方案。

输出格式：Markdown 表格。
```

## 不确定点

1. 提示词长度是否合适？5 个中最长约 200 字，最短约 150 字。
2. 提示词 5 与 A2 审查表有功能重叠。

## 风险点

1. 用户可能直接复制但不理解分类标准，导致输出质量不稳定。
2. 提示词 2 可能让用户过度降强。

## 需要 L3/L4 复核的问题

1. 提示词 5 与 A2 审查表：合并还是并存？——需 L3 裁定。
2. 提示词是否需增加"适用模型等级"标注？——需 L3 裁定。
