# 差结构学习法 · AI 协作判断力工具包

> 首次使用入口 | 版本：v0.1 试发布稿 | 第一轮 L3/L4 复核通过，待少量小修后正式发布

---

## 这是什么

一套帮助你和 AI 更稳地判断复杂问题的工具包。包含：

- **10 张方法卡**：每张解决一个判断问题
- **1 张 AI 输出审查表**：检查 AI 回答是否可靠
- **5 个 AI 提示词**：可直接复制给 AI 使用
- **2 个判断模板**：差异判断表 + 行动复盘表

核心判断顺序：

```text
先看差异 → 再看边界 → 再看落差 → 再看耗散 → 再看结构 → 最后看能不能验证
```

## 一、10 张方法卡

### 卡片 1：四词拆开法

**核心**：差异、差结构、落差、可用落差是四个层级，不能混写。

**动作**：遇到"不同"时，依次追问：这只是差异吗？有排布方式吗？有方向性和强弱吗？系统能调用它吗？

**误区**：把"差异"直接写成"落差"，跳过边界和耦合就说"可以利用"。

**AI 提示词**：`请帮我检查以下文本中，"差异""差结构""落差""可用落差"四个词是否被混用。对每个混用位置，说明应该用哪个层级。`

---

### 卡片 2：可用落差判据式

**核心**：可用落差 ≈ 势差 × 耦合 × 路径 × 效率，任一为零则不可用。

**动作**：对任何"机会"，检查四因子：势差多大？能接上吗？有通道吗？效率够吗？

**误区**：有差异就认为可用。

**AI 提示词**：`请用"势差×耦合×路径×效率"框架分析以下差异是否可用。对每个因子给出0-5分评估，并指出哪个因子是瓶颈。`

---

### 卡片 3：总量≠可用量

**核心**：能量总量守恒，但可用能会下降；资源多≠可用资源多。

**动作**：评估资源时区分两个问题：总量多少？其中多少还能被调用？

**误区**：以为信息量大就等于可用信息多。

**AI 提示词**：`请帮我区分以下论述中的"总量"和"可用量"。对每项资源，分别评估总量和可被调用的比例。`

---

### 卡片 4：落差来源五分法

**核心**：可用落差来自五个来源：初始条件、边界条件、约束结构、外部输入、涨落放大。

**动作**：诊断"为什么没有落差"时，逐一检查五个来源。

**误区**：只看"外部机会"，忽略约束保存的落差。

**AI 提示词**：`请从五个来源（初始条件、边界条件、约束结构、外部输入、涨落放大）分析当前系统为什么缺乏可用落差，或可用落差从何而来。`

---

### 卡片 5：问题化判断法

**核心**：问题 = 差异进入主体目标、模型和行动接口后的可用认知落差。

**动作**：判断差异是否值得处理，检查三要素：有目标吗？能识别吗？有行动路径吗？

**误区**：把所有"不知道"都当成问题。

**AI 提示词**：`请判断以下差异是否构成"问题"。按三要素分析：1)目标状态是什么？2)主体能否识别？3)有无行动路径？缺哪项就标明。`

---

### 卡片 6：学习链诊断法

**核心**：学习链条：差异→问题→行动→反馈→模型更新，断链即失败。

**动作**：诊断学习失败时，逐节检查链条，断在哪节修哪节。

**误区**：以为"学了"等于"会了"。

**AI 提示词**：`请用"差异→问题→行动→反馈→模型更新"链条诊断我的学习瓶颈。对每个环节判断是否断裂，并给出修复建议。`

---

### 卡片 7：耗散三分类

**核心**：耗散分正向（生成结构）、维持性（保持运转）、负向（消耗无结构），耗散≠进步。

**动作**：对每项行动问：留下了可复用结构吗？维持了必要运转吗？消耗了资源但没留下结果吗？

**误区**：把忙碌当成成长。

**AI 提示词**：`请把以下行动清单按正向耗散（生成结构）、维持性耗散（保持运转）、负向耗散（消耗无结构）分类，并计算大致比例。`

---

### 卡片 8：结构沉淀五条件

**核心**：行动结果变成结构需要：可识别、可维持、可调用、可复制、可反馈。

**动作**：对行动结果逐一检查五条件，缺哪项补哪项。

**误区**：以为"做过了"就有结构。

**AI 提示词**：`请用"可识别、可维持、可调用、可复制、可反馈"五条件检查表评估以下行动结果是否沉淀为结构。对每个条件给出通过/未通过判断。`

---

### 卡片 9：吸收系数检查

**核心**：外置结构不自动增强主体，同一结构对不同主体的作用取决于吸收系数。

**动作**：评估外部工具时，检查八项能力：提问、分解、判断边界、反方验证、工作流设计、输出保存、反馈修正、主体性。

**误区**：买书=理解；使用AI=获得判断力。

**AI 提示词**：`请帮我评估我对以下工具/资源的吸收系数。按八项能力逐一打分，并给出提升方案。`

---

### 卡片 10：差异可用十问

**核心**：判断差异是否可用，依次回答十个问题，答不出则最多只是启发。

**动作**：1被比较对象？2差异轴？3参照状态？4边界？5耦合？6释放路径？7损耗？8反馈？9封存？10失败条件？

**误区**：跳过检查直接下结论。

**AI 提示词**：`请用"差异可用十问"检查表分析以下差异是否可用。对每个问题给出明确回答，答不出的标明为瓶颈。`

## 二、AI 输出审查表

拿到一段 AI 回答后，逐项填写：

| # | 字段 | 填写 |
|---|---|---|
| 1 | 事实是什么 |  |
| 2 | 推断是什么 |  |
| 3 | 类比是什么 |  |
| 4 | 建议是什么 |  |
| 5 | 证据在哪里 |  |
| 6 | 边界在哪里 |  |
| 7 | 失败条件是什么 |  |
| 8 | 下一步如何验证 |  |
| 9 | 是否可以直接行动 |  |
| 10 | 需要追问 AI 什么 |  |

**快速版（3步）**：①分三层（事实/推断/类比）②找边界（什么条件下不成立）③写验证（用什么动作检验）

## 三、5 个 AI 提示词

### 1. 判断差异在哪里

```text
请帮我分析以下问题中真正的差异在哪里。
1. 被比较的两个对象或状态是什么？
2. 差异轴是什么？
3. 这个差异是否形成落差？
4. 这个落差是否可用？
5. 如果不可用，缺哪个条件？
6. 给出一个可验证的下一步动作。
```

### 2. 检查观点是否过强

```text
请帮我检查以下观点是否过强。
1. 有没有把桥接解释写成已证明结论？
2. 有没有把跨领域类比写成同一机制？
3. 有没有缺少边界的全称判断？
4. 有没有把工作假说写成稳结论？
5. 有没有把局部有效写成普遍规律？
对每个问题：标出原句、说明风险、给出降强方向。
```

### 3. 把想法压成可验证行动

```text
请帮我把以下想法压成可验证行动。
1. 一句话说清核心判断；
2. 判断层级：事实/推断/类比/假说/建议；
3. 如果对，最直接的验证动作；
4. 如果错，什么结果能看出来；
5. 最小成本估算；
6. 不超过30分钟的最小验证动作。
```

### 4. 复盘行动有没有留下结构

```text
请帮我复盘以下行动是否留下了可复用结构。
检查：可识别？可维持？可调用？可复制？可反馈？
如果五项都未通过，可能只是一次性耗散。
最后判断：正向/维持性/负向耗散？下次如何增加结构沉淀？
```

### 5. 把 AI 输出按四类重排

```text
请把以下 AI 输出按四类重排：边界、证据、假设、残差。
- 边界：什么条件下成立/不成立
- 证据：有来源或数据支撑的部分
- 假设：推断、类比或未检验的桥接
- 残差：当前无法判断的部分
对每条假设给出检验动作，对每条残差说明需要什么材料。
```

## 四、判断模板

### 差异判断表

| # | 问题 | 回答 |

| 2 | 差异轴是什么？ |  |

### 行动复盘表

| # | 项目 | 通过/部分/未通过 |
|---|---|---|
| 1 | 结果可识别？ |  |
| 2 | 结果可维持？ |  |
| 3 | 结果可调用？ |  |
| 4 | 结果可复制？ |  |
| 5 | 结果可反馈？ |  |

耗散判断：□ 正向（生成结构） □ 维持性（保持运转） □ 负向（消耗无结构）

残差：这次留下了什么？还有什么没解决？下次怎么改？

## 使用建议

1. **第一次用**：先看卡片 1（四词拆开法）和卡片 10（差异可用十问），用差异判断表试一次。
2. **判断 AI 回答**：用审查表或提示词 5。
3. **复盘行动**：用行动复盘表或提示词 4。
4. **遇到新问题**：用提示词 1 先判断差异在哪里。
