# 附录 L2 第二轮产出：评论回写接口定义

> 任务编号：R2-P0-04 | 任务等级：T2/T3 | 执行模型：L2 | 回写判定：只能作为候选
> 输入材料：外部反馈信 + 附录_残差登记卡、版本更新单与反馈回写表
> 执行日期：2026-05-02

---

## 一、评论回写最少字段

一条评论要进入回写流程，至少需要以下字段：

| # | 字段 | 必填 | 说明 |
|---|---|---|---|
| 1 | 原始内容 | 是 | 评论原文或摘要 |
| 2 | 来源 | 是 | 站点评论 / 社交媒体 / 邮件 / 私聊 / 外部反馈信 / 其他 |
| 3 | 来源日期 | 是 | 评论发布或收到的时间 |
| 4 | 评论类型 | 是 | 事实纠正 / 概念质疑 / 方法建议 / 案例补充 / 误读反馈 / 情感共鸣 / 其他 |
| 5 | 命中章节 | 是 | 评论指向的章节或专题 |
| 6 | 处理动作 | 是 | 吸收 / 补桥 / 降强 / 转专题 / 登记残差 / 明确拒绝 / 暂挂 |
| 7 | 处理依据 | 是 | 为什么选择这个动作 |
| 8 | 回写位置 | 否 | 如果需要回写，写到哪个文件哪个位置 |
| 9 | 可信状态 | 是 | 已验证 / 待验证 / 无法验证 |

---

## 二、评论来源分级

| 等级 | 来源类型 | 说明 | 默认处理 |
|---|---|---|---|
| A | 外部反馈信 | 独立审阅者写的结构化反馈 | 优先处理，必须逐条回应 |
| B | 站点评论 | 读者在文章下方留下的评论 | 筛选后处理 |
| C | 社交媒体讨论 | 微博、知乎、X 等平台的讨论 | 摘要后处理 |
| D | 私聊/邮件 | 个人沟通中的反馈 | 记录后处理 |
| E | 匿名反馈 | 无法验证来源的反馈 | 只进入候选池，不直接回写 |

---

## 三、评论只能进入候选池的情况

以下情况的评论不能直接回写正文，只能进入候选池：

1. 来源等级为 E（匿名反馈）且无法验证
2. 评论内容含事实主张但无任何出处
3. 评论与已有评论冲突且无法判断谁对
4. 评论涉及核心术语定义修改（需 L4 裁定）
5. 评论要求改变项目方向或定位（需 L4 裁定）
6. 评论含情绪攻击但无实质内容（暂挂）
7. 评论含商业推广或无关内容（明确拒绝）

## 四、空接口说明（暂无站点评论数据时）

当前状态：无站点评论数据，评论回写接口为空。

空接口期间的替代方案：

| 替代来源 | 操作方式 | 等效等级 |
|---|---|---|
| 外部反馈信 | 已收录（见附录_L2产出_外部反馈信_对照分析.md） | A |
| 作者自审 | 作者阅读后标注问题 | A |
| L2 自审 | L2 按自审七步执行 | B |
| 同行评审 | 邀请同行阅读并反馈 | A |

空接口保留的最低字段要求：

```markdown
|---|---|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |  |  |
```

## 五、外部反馈信、残差登记与版本更新的关系

```text
外部反馈信 ──→ 评论回写接口 ──→ 处理动作（六选一）
                                    │
                                    ├── 吸收/补桥/降强 ──→ 版本更新单
                                    ├── 转专题 ──→ 新专题文件
                                    ├── 登记残差 ──→ 残差登记卡
                                    ├── 明确拒绝 ──→ 记录但不回写
                                    └── 暂挂 ──→ 候选池

残差登记卡 ──→ 状态变化 ──→ 版本更新单
版本更新单 ──→ 回写同步 ──→ 术语表/目录/站点入口
```

三者关系说明：

| 概念 | 触发条件 | 输出 | 与其他两者的关系 |
|---|---|---|---|
| 评论回写 | 收到外部输入 | 处理动作 + 回写位置 | 评论回写可能触发残差登记或版本更新 |
| 残差登记 | 发现主线未吸收的重复出现 | 残差登记卡（名称/节点/类型/动作/状态） | 残差状态变化触发版本更新 |
| 版本更新 | 主线/术语/目录/站点入口发生改动 | 版本更新单（改动内容/位置/原因/同步检查） | 版本更新必须同步所有相关位置 |

## 不确定点

1. 评论类型分类是否完整？"误读反馈"是否应单独列出？
2. 来源分级中，"L2 自审"是否应视为 B 级？L2 产出本身需要 L3 复核。
3. 空接口期间，是否应先由 L2 执行一轮自审来填充接口？

## 风险点

1. 评论回写可能被误用为"所有评论都必须回写"，但实际大部分评论只需暂挂或明确拒绝。
2. 来源分级可能被忽略，导致低可信度评论直接进入正文。
3. 空接口期间如果没有任何评论进入，接口可能被遗忘。

## 需要 L3/L4 复核的问题

1. 评论类型分类是否完整？——需 L3 评估。
2. 站点评论数据的正式接入方式？——需 L4 指定。
3. 是否先由 L2 执行自审填充空接口？——需 L3 裁定。
4. 评论回写接口的回写位置？——需 L4 裁定。
