# 任务模板

当技能已经触发，而你又想要一个更贴近当前任务的执行模板时，读这个文件。

- 如果任务直接指向当前仓库的书稿、`README.md`、`AI_ENTRY.md` 或站点文案，先读 [repo-bridge.md](repo-bridge.md) 再套下面的模板。
- 如果用户只要最小可复用骨架，优先直接复用各节里的“建议输出形状”。

## 1. 审理论或文章

当用户要你做批评、审稿、加固时，用这个模板。

重点检查：

- 比较协议缺在哪
- 不同差异轴混在哪
- 类比是从哪一步开始冒充机制的
- 结论是从哪里开始过早变成普遍说法的
- 证明、实验和修辞是从哪里开始混写的

建议输出形状：

1. 最有价值的想法
2. 最大的过强处
3. 缺的桥
4. 改写方向
5. 失败问题

强触发示例：

- “审这份理论草稿，不要让它靠宏大统一语言蒙混过关。”
- “检查这篇文章，找出类比悄悄变成机制的地方。”
- “告诉我这个框架从哪里开始不再有用，而变得不可证伪。”
- “像一个苛刻审稿人一样读这篇文章，把缺失的边界指出来。”

## 2. 判断是否转入差结构分析 / 做定性

当用户问“这个问题该怎么分析”“真正差异在哪里”“先定性一下”时，用这个模板。

重点检查：

- 问题是否出现残差、症状迁移或反复失败
- 是否存在对象、层次、边界、生命周期或判准混写
- 是否需要先固定比较协议，而不是直接给结论
- 差异主要属于性质、程度、结构、位置、生命周期、反馈、边界、判准还是证据强度

建议输出形状：

1. 是否值得转入差结构分析
2. 触发理由
3. 比较对象
4. 差异轴
5. 差异定性
6. 当前只能保留到什么强度
7. 失败条件或残差

强触发示例：

- “这个问题适合用差结构方法吗？”
- “先别急着解决，帮我判断真正差异在哪里。”
- “给这个现象定性，它到底是边界问题、反馈问题还是判准问题？”
- “这个争论是不是把不同层次混在一起了？”

## 3. 改段落或提示词

当用户让你“优化”“收紧”“别那么虚”时，用这个模板。

按这个顺序做：

1. 先说对象
2. 再说差异轴
3. 再说边界
4. 把层次拆开
5. 降强没支撑的说法
6. 补上最小有用失败条件

建议输出形状：

1. 这一版最需要收紧的地方
2. 收紧后的改写
3. 如果要保留更强野心，可单列成桥接解释或工作假说
4. 最小失败条件或降级条件

强触发示例：

- “把这段话改得更严谨一点，别再像宇宙宣言。”
- “收紧这个提示词，让 AI 别再动不动就普遍化。”
- “把这个说法改得更可检验，但别丢掉主意。”
- “重写这一段，让证明、实验和类比别再混在一起。”

## 4. 诊断系统、习惯、流程或团队

当用户想知道某物为什么稳定、失稳、嘈杂或退化时，用这个模板。

重点追问：

- 被维持的结构到底是什么
- 它依赖什么输入或刷新
- 现在主导的是哪种噪声或泄漏
- 是哪条边界坏了或太松
- 哪个变量正在逼近阈值
- 怎样改能改善稳定性，但又不假装一次性解决全部问题

建议输出形状：

1. 被维持的结构
2. 维持它的输入、边界或刷新
3. 当前主导的噪声、泄漏或耗散
4. 最接近的阈值或失败点
5. 最小修复动作
6. 仍未判明的局部状态

强触发示例：

- “为什么这个团队流程每过几周就开始退化？”
- “用边界和稳态分析，诊断为什么我的学习习惯总是立不住。”
- “这个系统到底漏掉了什么，真正需要刷新的又是什么？”
- “解释一下，为什么这个流程纸面上稳定，现实里却总失败。”

## 5. 设计研究问题

当用户有一个大想法，但需要把它压成可检验问题时，用这个模板。

把想法压成：

- 一个局部对象
- 一到两条差异轴
- 一个观察或测量方案
- 一个基线模型
- 一个失败条件
- 一个测试后保留或丢弃它的理由

建议输出形状：

1. 局部对象
2. 一到两条差异轴
3. 观察或测量方案
4. 基线模型
5. 失败条件
6. 结果如何升级、保留或降级

强触发示例：

- “把这个宽泛的哲学主张压成一个有边界的研究问题。”
- “帮我把这个想法操作化，给出可观测变量和基线。”
- “让这个假说变得能失败，而不只是看上去厉害。”
- “把这个概念框架压成一个小型测试计划。”

## 6. 搭一个更好的 AI 提示词或工作流

当另一个 AI 容易发散、讨好或过度泛化时，用这个模板。

要求它做到：

- 先识别任务类型
- 把比较对象说清楚
- 分开本体、证据和流程
- 桥接解释要明确标成桥接解释
- 残差要记下来，不要藏掉
- 明说什么会让它自己的输出被证伪或降级

建议输出形状：

1. 任务类型
2. 输入对象和边界
3. 输出分层
4. 禁止自动升级的规则
5. 残差记录位
6. 自证伪或降级条件

强触发示例：

- “重写这个系统提示词，让模型别再混淆理论、证据和建议。”
- “把这个审稿提示词改得更苛刻一点，专门抓过强和伪统一。”
- “帮我搭一个 AI 工作流，要求它记录残差，而不是假装所有问题都解决了。”
- “把这个松散的分析提示词压成一个结构化审查协议。”

## 7. 通用触发示例

下面这些都是很好的触发语句：

- “审这份理论草稿，但别让它滑成伪统一。”
- “这个问题该不该转成差结构分析？”
- “先帮我定性：真正差异在哪里？”
- “用这套方法分析，为什么我的学习计划总是崩。”
- “重写这个提示词，让 AI 分开证明、实验和类比。”
- “把这个抽象哲学主张压成一个有边界的研究问题。”
- “比较这两个系统，但不要说它们字面上就是同一机制。”
- “诊断这个框架到底有没有真实失败条件，还是只有漂亮语言。”
- “把这个大想法改得更小、更锋利、更可操作。”
- “审一下这个流程，找出隐藏阈值、泄漏点和断裂边界。”
- “指出这个论证在哪些地方偷偷换了层次。”
