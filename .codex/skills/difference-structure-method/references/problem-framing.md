# 问题转入与差异定性

当任务不是已经明确要审稿、改写或诊断，而是在问“这个问题该怎么分析”“真正差异在哪里”“怎么定性”时，先读这个文件。

## 1. 什么时候转入差结构分析

普通问题不必都转成差结构分析。出现下面任一信号时，再转入：

- 同一个问题反复出现，局部修正只能移动症状
- 用户反馈指出“改了像没改”“方向不对”“换个角度”
- 讨论里有多个层次混在一起：事实、机制、认识、流程、价值判断互相替代
- 结论听起来比证据更大，或者一个概念开始解释一切
- 跨领域比较变成“其实都是同一种机制”
- 系统稳定性、退化、反馈、边界、生命周期或残差成为核心问题
- 争论卡在“是不是同一个东西”，但比较协议还没有固定

如果只是单点事实查询、明确代码错误、直接格式改写或已有标准领域方法，先不用这套方法。

## 2. 先定性，不先定论

定性不是给最终答案，而是判断“这个问题首先属于哪类差异”。先问：

- 对象差异：比较对象是不是拿错了
- 性质差异：两者是不是属于不同种类
- 程度差异：是不是同一轴上的强弱、多少、快慢、密度差
- 结构差异：元素差不多，但组织方式、连接方式或层级不同
- 位置差异：同一结构里处在不同位置、角色或通道
- 生命周期差异：创建、维持、失效、刷新和释放时机不同
- 边界差异：什么算内外、有效无效、同一不同还没定
- 反馈差异：反馈有没有回到正确层，还是只改变表面动作
- 判准差异：双方用的成功标准、失败标准或证据标准不同
- 证据强度差异：一个是证明，一个是观察，一个是类比，一个只是建议

定性完成后，再决定能否比较、能否归类、能否上升为结构。

## 3. 差异轴怎么定

一个好的差异轴要满足：

- 两边对象能放到同一比较协议里
- 轴上至少能说明方向、尺度或分档
- 边界改变时，判断会怎样改变
- 能说出什么观察会让这个轴失效

坏差异轴的信号：

- 只用一个大词命名，但没有对象、尺度或参照物
- 把多个轴压成一个轴，比如把快慢、真假、价值高低混在一起
- 只有赞美或批评功能，没有失败条件
- 发现反例后不停往“局部状态”里塞变量

## 4. 从差异到结构的升级条件

不要把差异自动升级为结构。至少满足下面几项，才可以更强地说“这里有结构”：

1. 差异不是偶发点，而是有排布方式
2. 排布方式在某个边界内重复出现
3. 有维持条件、资源输入、反馈或约束
4. 能说出失效、退化或断裂条件
5. 至少能和一个基线或替代解释比较

如果这些不够，只标成桥接解释或工作假说。
