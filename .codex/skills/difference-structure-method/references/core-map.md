# 核心图

当你只想抓住这套方法最短、可复用的骨架时，读这个文件。

## 1. 最稳的身份

- 当前最稳的身份：判断方法、研究程序或协作纪律
- 不是已经完成的总本体论
- 不是替代标准学科理论的通行证

## 2. 最短顺序

除非某个领域已经有更强的标准流程，否则优先用这个顺序：

1. 先判断是否需要转入差结构分析
2. 先认出哪里不同
3. 给差异定性：性质、程度、结构、位置、生命周期、边界、反馈、判准或证据强度
4. 固定差异轴
5. 固定尺度、边界、参照和局部状态
6. 看这些差异是怎么排布的
7. 看它们是否形成了可比较的不均或可用梯度
8. 看什么在传递、封存、稳定或降解它们
9. 看什么真的够资格叫结构或规则
10. 看失败会怎样出现

## 3. 层次拆分

下面几层尽量分开：

- 本体层：什么存在
- 结构 / 世界层：什么在系统里站住
- 认识层：某物如何被证明、观察、建模或确认
- 程序层：人或 AI 应该如何处理它

## 4. 必须守住的区分

- 差异是关系，不是实体
- 定性先于定论
- 桥接词不会自动升级成机制
- 形式证明不是经验归纳
- 物理定律确认不是“重复几次就向全局外推”
- 局部状态不是垃圾桶
- 门槛模板还不是普遍定律

## 5. 强度标签

需要时，把材料标成：

- 可稳妥保留的结论
- 桥接解释
- 工作假说
- 残差 / 开放问题

## 6. 常见改写动作

优先这样改：

- “X 解释一切” -> “X 是当前分析 Y 的入口”
- “所有规则都这样形成” -> “许多经验世界中的规则可以这样诊断”
- “差异生成结构” -> “差异帮助识别结构”
- “这就是同一种机制” -> “这几个案例可以按同一判断顺序比较”
- “这已经被证明” -> 明说它究竟是被证明、被拟合、被观察、被推断，还是只是被建议

## 7. 失败纪律

如果这套方法被用对了，最后至少应该逼出下面一项：

- 更紧的边界
- 更小的结论
- 更清楚的协议
- 一个基线比较
- 一个明确失败测试
- 一条残差记录，而不是继续把口号写大
